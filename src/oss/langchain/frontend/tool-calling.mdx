## 工具调用的工作原理

当 LangGraph Agent 决定需要外部数据时，它会在 AI 消息中发出一个或多个**工具调用**。每个工具调用包含：

- **name**：被调用的工具（例如 `"get_weather"`、`"calculator"`）
- **args**：传递给工具的结构化参数
- **id**：将调用与其结果关联的唯一标识符

Agent 运行时执行工具，结果以 `ToolMessage` 形式返回。`useStream` 钩子将所有这些统一为一个 `toolCalls` 数组，你可以直接渲染。

## 设置 useStream

第一步是将 `useStream` 连接到 Agent 后端。钩子返回响应式状态，包括一个随 Agent 流式输出实时更新的 `toolCalls` 数组。

:::python

定义一个与 Agent 状态 schema 匹配的 TypeScript 接口，并将其作为类型参数传递给 `useStream`，以获得类型安全的状态值访问。在下面的示例中，将 `typeof myAgent` 替换为你的接口名称：


## ToolCallWithResult 类型

`toolCalls` 数组中的每个条目都是一个 `ToolCallWithResult` 对象：

```ts
interface ToolCallWithResult {
  call: {
    id: string;
    name: string;
    args: Record<string, unknown>;
  };
  result: ToolMessage | undefined;
  state: "pending" | "completed" | "error";
}
```

| 属性 | 描述 |
| --- | --- |
| `call.id` | 与 AI 消息的 `tool_calls` 条目匹配的唯一 ID |
| `call.name` | 工具名称（例如 `"get_weather"`） |
| `call.args` | Agent 传递给工具的结构化参数 |
| `result` | `ToolMessage` 响应，在工具完成后可用 |
| `state` | 生命周期状态：运行中为 `"pending"`，成功为 `"completed"`，失败为 `"error"` |

## 过滤每条消息的工具调用

一条 AI 消息可能触发多个工具调用，而你的聊天中可能有多条 AI 消息。要在每条消息下渲染正确的工具卡片，通过将 `call.id` 与消息的 `tool_calls` 数组匹配来过滤：

```tsx
function Message({
  message,
  toolCalls,
}: {
  message: AIMessage;
  toolCalls: ToolCallWithResult[];
}) {
  const messageToolCalls = toolCalls.filter((tc) =>
    message.tool_calls?.find((t) => t.id === tc.call.id)
  );

  return (
    <div>
      <p>{message.content}</p>
      {messageToolCalls.map((tc) => (
        <ToolCard key={tc.call.id} toolCall={tc} />
      ))}
    </div>
  );
}
```

## 构建专门的工具卡片

不要直接显示原始 JSON，而是为每个工具构建专门的 UI 组件。使用 `call.name` 选择正确的卡片：

```tsx
function ToolCard({ toolCall }: { toolCall: ToolCallWithResult }) {
  if (toolCall.state === "pending") {
    return <LoadingCard name={toolCall.call.name} />;
  }

  if (toolCall.state === "error") {
    return <ErrorCard name={toolCall.call.name} error={toolCall.result} />;
  }

  switch (toolCall.call.name) {
    case "get_weather":
      return <WeatherCard args={toolCall.call.args} result={toolCall.result} />;
    case "calculator":
      return (
        <CalculatorCard args={toolCall.call.args} result={toolCall.result} />
      );
    case "web_search":
      return <SearchCard args={toolCall.call.args} result={toolCall.result} />;
    default:
      return <GenericToolCard toolCall={toolCall} />;
  }
}
```

```tsx
function WeatherCard({
```

### 加载和错误状态

始终处理 pending 和 error 状态，给用户清晰的反馈：

```tsx
function LoadingCard({ name }: { name: string }) {
  return (
    <div className="flex items-center gap-2 rounded-lg border p-4 animate-pulse">
      <Spinner />
      <span>Running {name}...</span>
    </div>
  );
}

function ErrorCard({ name, error }: { name: string; error?: ToolMessage }) {
  return (
    <div className="rounded-lg border border-red-300 bg-red-50 p-4">
      <h3 className="font-semibold text-red-700">Error in {name}</h3>
      <p className="text-sm text-red-600">
        {error?.content ?? "Tool execution failed"}
      </p>
    </div>
  );
}
```

## 类型安全的工具参数

如果你的工具使用结构化 schema 定义，你可以使用 `ToolCallFromTool` 实用类型来获得完全类型化的 `args`：

```ts
import { tool } from "@langchain/core/tools";
import { z } from "zod";

const getWeather = tool(async ({ location }) => { /* ... */ }, {
  name: "get_weather",
  description: "Get the current weather for a location",
  schema: z.object({
    location: z.string().describe("City name"),
  }),
});

type WeatherToolCall = ToolCallFromTool<typeof getWeather>;
// WeatherToolCall.call.args is now { location: string }
```

<Tip>
使用 `ToolCallFromTool` 可以获得编译时安全。如果工具 schema 变更，你的 UI 组件会立即标记类型错误。
</Tip>

## 在流式文本中内联渲染工具调用

工具调用通常与流式文本交错到达。`useStream` 钩子保持 `toolCalls` 与流同步，因此在 Agent 发出调用后、工具完成执行前，pending 卡片就会立即出现。

这意味着用户会看到：

1. AI 的文本在流式传入
2. 工具调用发出的瞬间出现加载卡片
3. 工具完成后卡片更新显示结果

<Note>
工具调用是原地更新的。相同的 `call.id` 从 `"pending"` 过渡到 `"completed"`（或 `"error"`），因此你的 UI 用新状态重新渲染相同的组件。
</Note>

## 处理多个并发工具调用

Agent 可以并行调用多个工具。`toolCalls` 数组会同时包含多个 `state: "pending"` 的条目。每个独立完成，因此你的 UI 应优雅处理部分完成的情况：

```tsx
function ToolCallList({ toolCalls }: { toolCalls: ToolCallWithResult[] }) {
  const pending = toolCalls.filter((tc) => tc.state === "pending");
  const completed = toolCalls.filter((tc) => tc.state === "completed");

  return (
    <div className="space-y-2">
      {completed.map((tc) => (
        <ToolCard key={tc.call.id} toolCall={tc} />
      ))}
      {pending.map((tc) => (
        <LoadingCard key={tc.call.id} name={tc.call.name} />
      ))}
    </div>
  );
}
```

## 最佳实践

构建工具调用 UI 时请遵循以下准则：

- **始终处理所有三种状态**：`pending`、`completed` 和 `error`。用户不应看到空白卡片。
- **安全解析结果**。工具结果以字符串形式到达。将 `JSON.parse()` 包裹在 try/catch 中，解析失败时显示回退内容。
- **提供通用回退**。不是每个工具都需要专门的卡片。为未知工具名称渲染可折叠的 JSON 视图。
- **在加载期间显示工具名称和参数**。用户想知道 Agent *在做什么*，即使结果还没到达。
- **保持卡片紧凑**。工具卡片与聊天消息内联展示。避免用过大的组件淹没对话。
