加入和重新连接让你可以在不停止 Agent 的情况下断开正在运行的 Agent 流，然后稍后重新连接。Agent 在客户端离开期间继续在服务端执行，你可以从离开的地方精确恢复流。

## 为什么需要加入和重新连接？

传统的流式 API 将客户端和服务器紧密耦合：如果客户端断开，流就丢失了。加入和重新连接打破了这种耦合，支持以下重要模式：

- **网络中断**：在基站或 Wi-Fi 网络之间切换的移动用户可以无缝恢复
- **页面导航**：用户从聊天页面导航离开后返回，不丢失进度
- **移动端后台化**：被操作系统挂起的应用在前台恢复时可以重新连接流
- **长时间运行的任务**：执行多分钟操作（研究、代码生成、数据分析）的 Agent，用户无需保持页面打开
- **多设备切换**：在手机上开始对话，在桌面上重新连接

## 核心概念

加入/重新连接模式涉及三个关键机制：

| 方法/选项 | 用途 |
|---|---|
| `stream.stop()` | 断开客户端与流的连接，不停止 Agent |
| `stream.joinStream(runId)` | 通过运行 ID 重新连接到现有流 |
| `onDisconnect: "continue"` | 提交选项，告诉服务器在客户端断开后继续运行 |
| `streamResumable: true` | 提交选项，使流可以稍后被重新加入 |

<Note>
`stream.stop()` 与取消运行有本质区别。停止只断开**客户端**。Agent 继续在服务端处理。要实际取消 Agent 的执行，应使用中断或取消机制。
</Note>

## 使用可恢复选项提交

提交消息时，传递 `onDisconnect: "continue"` 和 `streamResumable: true` 以启用加入/重新连接流程：

```ts
stream.submit(
  { messages: [{ type: "human", content: text }] },
  {
    onDisconnect: "continue",
    streamResumable: true,
  }
);
```

| 选项 | 默认值 | 描述 |
|---|---|---|
| `onDisconnect` | `"cancel"` | 客户端断开时的行为。`"continue"` 保持 Agent 运行；`"cancel"` 停止它。 |
| `streamResumable` | `false` | 当为 `true` 时，服务器保留流状态以便客户端稍后重新加入。 |

<Tip>
始终同时使用两个选项。设置 `onDisconnect: "continue"` 而不设置 `streamResumable: true` 意味着 Agent 继续运行但你无法重新连接流来查看其输出。
</Tip>

## 断开流连接

调用 `stream.stop()` 断开客户端。Agent 继续在服务端处理。

```ts
stream.stop();
```

调用 `stop()` 后：
- `stream.isLoading` 变为 `false`
- 消息列表保留到断开点为止接收到的所有消息
- Agent 继续在服务器上运行
- 在重新连接之前不会收到新消息

## 重新连接流

使用保存的运行 ID 调用 `stream.joinStream(runId)` 重新连接：

```ts
stream.joinStream(savedRunId);
```

重新连接后：
- `stream.isLoading` 再次变为 `true`
- 断开期间生成的所有消息会被传递
- 新的流式消息实时恢复
- 如果 Agent 已经完成，你会立即收到最终状态

## 构建连接状态指示器

视觉指示器帮助用户了解他们是否正在活跃地接收 Agent 的更新。

```tsx

  return (
    <div className="connection-status">
      <span

        {connected ? "Connected" : "Disconnected"}

    </div>

}
```

## 断开和重新连接控件

提供明确的断开和重新连接按钮，让用户完全控制：

```tsx
function ChatControls({ stream, savedRunId, isConnected }) {
  const [input, setInput] = useState("");

  const handleSend = () => {
    if (!input.trim()) return;
    stream.submit(
      { messages: [{ type: "human", content: input.trim() }] },
      { onDisconnect: "continue", streamResumable: true }
    );
    setInput("");
  };

  return (
    <div className="controls">
      <div className="input-row">
        <input
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Type a message..."
          onKeyDown={(e) => e.key === "Enter" && handleSend()}
        />
        <button onClick={handleSend}>Send</button>
      </div>

      <div className="stream-controls">
        {isConnected ? (
          <button onClick={() => stream.stop()} className="disconnect-btn">
            Disconnect
          </button>
        ) : (
          savedRunId && (
            <button
              onClick={() => stream.joinStream(savedRunId)}
              className="rejoin-btn"
            >
              Rejoin stream
            </button>
          )
        )}
      </div>
    </div>
  );
}
```

## 持久化运行 ID

对于跨会话重新连接（如用户关闭浏览器后返回），将运行 ID 持久化到存储中：

```ts
const stream = useStream<typeof myAgent>({
  apiUrl: "http://localhost:2024",
  assistantId: "join_rejoin",
  onCreated(run) {
    localStorage.setItem("activeRunId", run.run_id);
  },
});

// On page load, check for an active run
const existingRunId = localStorage.getItem("activeRunId");
if (existingRunId) {
  stream.joinStream(existingRunId);
}
```

<Info>
当运行完成时应清理持久化的运行 ID。监听流结束并移除存储的 ID，以避免尝试重新连接已完成的运行。
</Info>

## 错误处理

如果运行已过期、被删除或服务器已重启，重新连接可能失败。优雅地处理这些情况：

```ts
try {
  stream.joinStream(savedRunId);
} catch (error) {
  console.error("Failed to rejoin stream:", error);
  // Clear stale run ID and inform the user
  setSavedRunId(null);
  localStorage.removeItem("activeRunId");
}
```

```tsx
function JoinRejoinChat() {
  const [savedRunId, setSavedRunId] = useState<string | null>(null);

    assistantId: "join_rejoin",
    onCreated(run) {
      setSavedRunId(run.run_id);
    },
  });

  const isConnected = stream.isLoading;

  const handleSend = () => {
    if (!input.trim()) return;

  return (

      <header>
        <h2>Join & Rejoin Demo</h2>
        <ConnectionStatus connected={isConnected} />
      </header>

      <div className="messages">
        {stream.messages.map((msg, i) => (
          <MessageBubble key={i} message={msg} />
        ))}
      </div>

      <div className="controls">
        <form onSubmit={(e) => { e.preventDefault(); handleSend(); }}>
          <input
            value={input}
            onChange={(e) => setInput(e.target.value)}
            placeholder="Type a message..."
          />
          <button type="submit">Send</button>
        </form>

        <div className="stream-actions">
          {isConnected ? (
            <button onClick={() => stream.stop()}>
              Disconnect
            </button>
          ) : (
            savedRunId && (
              <button onClick={() => stream.joinStream(savedRunId)}>
                Rejoin stream
              </button>
            )
          )}
        </div>
      </div>
    </div>
  );
}
```

## 最佳实践

- **始终保存运行 ID**：没有它，重新连接是不可能的。同时使用组件状态和持久化存储以增强韧性。
- **展示清晰的连接状态**：用户应始终知道他们是在接收实时更新还是查看快照。
- **在可见性变化时自动重新连接**：使用 Page Visibility API 在用户返回标签页时自动重新连接。
- **设置合理的超时**：如果重新连接尝试耗时过长，改为获取线程历史。
- **清理已完成的运行**：Agent 完成后移除持久化的运行 ID，避免过期的重新连接尝试。
