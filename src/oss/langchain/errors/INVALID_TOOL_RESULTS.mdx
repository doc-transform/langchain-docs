---
title: INVALID_TOOL_RESULTS
---

<Note>
    目前仅在 `langchainjs`（JavaScript/TypeScript）中使用。
</Note>

当在工具调用操作中向模型传入不匹配、不赢或多余的 @[`ToolMessage`] 对象时，会触发此错误。

该错误源于一个基本要求：包含 `tool_calls` 的 assistant 消息后面必须紧跟对每个 `tool_call_id` 做出响应的工具消息。

当模型返回包含工具调用的 @[`AIMessage`] 时，你必须为每个工具调用提供恰好一个对应的 @[`ToolMessage`]，且 `tool_call_id` 必须匹配。

## 常见原因

* **响应不赢**：如果模型请求执行两个工具调用，但你只提供了一个响应消息，模型将拒绝这个不完整的消息链
* **重复响应**：为同一个 tool call ID 提供多个 @[`ToolMessage`] 对象会被拒绝，同样，ID 不匹配也会导致拒绝
* **孤立的工具消息**：在没有包含工具调用的 @[`AIMessage`] 的情况下发送 @[`ToolMessage`]，违反了协议要求

:::python
以下是一个典型的问题模式：

```python
# 模型请求了两个工具调用
response_message.tool_calls  # 返回 2 个调用

# 但只提供了一个 ToolMessage
chat_history.append(ToolMessage(
    content=str(tool_response),
    tool_call_id=tool_call.get("id")
))

model_with_tools.invoke(chat_history)
```
:::

:::js
以下是一个典型的问题模式：

```javascript
// 模型请求了两个工具调用
responseMessage.tool_calls // 返回 2 个调用

// 但只提供了一个 ToolMessage
chatHistory.push({
  role: "tool",
  content: toolResponse,
  tool_call_id: responseMessage.tool_calls[0].id
});

await modelWithTools.invoke(chatHistory); // 报错：INVALID_TOOL_RESULTS
```
:::

## 排查方法

要解决此错误：

* **检查配对数量**：确保每个 @[`AIMessage`] 中的工具调用都有对应的一个 @[`ToolMessage`]
* **验证 ID**：确认每个 `ToolMessage.tool_call_id` 与实际的工具调用标识符一致
