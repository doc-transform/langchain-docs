当[提示词模板](https://github.com/langchain-ai/langchain/blob/v0.3/docs/docs/concepts/prompt_templates.mdx)接收到缺失或无效的输入变量时，会触发此错误。

:::js
一种容易被忽视的触发方式是在提示词模板中直接嵌入 JSON 对象：

```typescript
import { PromptTemplate } from "@langchain/core/prompts";
import { ChatOpenAI } from "@langchain/openai";

const prompt = PromptTemplate.fromTemplate(`You are a helpful assistant.

Here is an example of how you should respond:

{
  "firstName": "John",
  "lastName": "Doe",
  "age": 21
}

Now, answer the following question:

{question}`);
```

你可能认为上面的提示词模板只需要一个名为 question 的输入变量，但 JSON 对象中的花括号（`{`）由于没有转义，会被解释为额外的变量。正确做法是使用双花括号进行转义，如下所示：

```typescript
import { PromptTemplate } from "@langchain/core/prompts";
import { ChatOpenAI } from "@langchain/openai";

const prompt = PromptTemplate.fromTemplate(`You are a helpful assistant.

Here is an example of how you should respond:

{{
  "firstName": "John",
  "lastName": "Doe",
  "age": 21
}}

Now, answer the following question:

{question}`);
```
:::


## 排查方法

要解决此错误，你可以：

1. 检查提示词模板的正确性。使用 f-string 格式时，确保花括号已正确转义：
   * 在 f-string 中使用 `{{` 来表示单个花括号
   * 在 f-string 中使用 `{{{{` 来表示双花括号
2. 使用 `MessagesPlaceholder` 组件时，确认传入的是消息数组或消息类对象。如果使用简写元组形式，需要用花括号包裹变量名，如 `["placeholder", "{messages}"]`
3. 通过 [LangSmith](/langsmith/home) 或日志记录检查提示词模板实际接收到的输入，验证其是否符合预期
4. 如果提示词来源于 LangChain [Prompt Hub](https://smith.langchain.com/prompts)，请先用示例输入单独测试该提示词，确保其按预期工作
