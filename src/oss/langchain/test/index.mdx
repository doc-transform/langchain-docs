---

Agent 应用让 LLM 自主决定下一步操作来解决问题。这种灵活性非常强大，但模型的黑盒特性使得很难预测修改某个部分会如何影响整个 Agent。要构建生产级 Agent，全面的测试至关重要。

测试 Agent 有以下几种方法：

- **单元测试** 使用内存中的假对象隔离测试 Agent 的小型确定性组件，以便快速、确定性地断言精确行为。
- **集成测试** 使用真实网络调用测试 Agent，确认组件协同工作、凭证和模式匹配、延迟可接受。
- **评估（Evals）** 使用评估器评估 Agent 的执行轨迹，可以通过确定性匹配或 LLM 评判实现。

Agent 应用通常更依赖集成测试，因为它们串联多个组件，并且必须处理 LLM 非确定性特性带来的不稳定性。
