LangChain 的发展基于以下几个核心信念：

- 大语言模型（LLM）是一项强大的新技术。
- LLM 与外部数据源结合时能发挥更大的价值。
- LLM 将重塑未来应用的形态——未来的应用将越来越具备 Agent 特性。
- 这场变革仍处于非常早期的阶段。
- 虽然构建 Agent 应用的原型很容易，但要构建可靠到能投入生产的 Agent，依然极具挑战。

围绕这些认知，LangChain 聚焦于两个核心方向：

<Steps>
    <Step title="让开发者能够使用最好的模型进行构建">
        不同的模型提供商有各自不同的 API，包括不同的模型参数和消息格式。
        标准化模型的输入和输出是核心工作之一，使开发者可以轻松切换到最新的前沿模型，避免厂商锁定。
    </Step>
    <Step title="让模型能够轻松编排与数据和计算交互的复杂流程">
        模型不应仅被用于*文本生成*——它们还应被用于编排与其他数据交互的更复杂流程。LangChain 让定义 LLM 可以动态调用的[工具（Tools）](/oss/langchain/tools)变得简单，同时也帮助处理和访问非结构化数据。
    </Step>
</Steps>

## 发展历程

鉴于该领域日新月异的变化，LangChain 也在持续演进。以下是 LangChain 多年来的发展简史，展示了它如何伴随 LLM 应用构建范式的演变而不断进化：

<Update label="2022-10-24" description="v0.0.1">
    在 ChatGPT 发布前一个月，**LangChain 作为 Python 包正式发布**。它包含两个核心组件：

    - LLM 抽象层
    - "链（Chains）"——即针对常见用例的预定义计算步骤。例如 RAG：先执行检索步骤，再执行生成步骤。

    LangChain 的名称来源于"Language"（语言模型）和"Chains"（链）。
</Update>

<Update label="2022-12">
    LangChain 添加了首批通用 Agent。

    这些通用 Agent 基于 [ReAct 论文](https://arxiv.org/abs/2210.03629)（ReAct 即 Reasoning and Acting 的缩写）。它们使用 LLM 生成代表工具调用的 JSON，然后解析该 JSON 以确定调用哪些工具。
</Update>

<Update label="2023-01">
    OpenAI 发布了"Chat Completion" API。

    此前，模型接收字符串输入并返回字符串。在 Chat Completions API 中，模型演进为接收消息列表并返回消息。其他模型提供商纷纷跟进，LangChain 也随之更新以支持消息列表。
</Update>

<Update label="2023-01">
    LangChain 发布 JavaScript 版本。

    LLM 和 Agent 将改变应用的构建方式，而 JavaScript 是应用开发者的主力语言。
</Update>

<Update label="2023-02">
    **LangChain Inc. 围绕开源 LangChain 项目正式成立公司**。

    公司的核心目标是"让智能体无处不在"。团队意识到，虽然 LangChain 是关键一环（它让使用 LLM 变得简单），但还需要其他配套组件。
</Update>

<Update label="2023-03">
    OpenAI 在其 API 中发布了"函数调用（Function Calling）"功能。

    这使得 API 能够显式生成代表工具调用的结构化数据。其他模型提供商相继跟进，LangChain 也更新为将此作为工具调用的首选方式（而非解析 JSON）。
</Update>

<Update label="2023-06">
    **LangSmith 作为 LangChain Inc. 的闭源平台发布**，提供可观测性和评估能力。

    构建 Agent 的核心难题在于使其足够可靠，而 LangSmith 通过提供可观测性和评估功能来解决这一需求。LangChain 随之更新以与 LangSmith 无缝集成。
</Update>

<Update label="2024-01" description="v0.1.0">
    **LangChain 发布 0.1.0 版本**，这是首个非 0.0.x 版本。

    行业从原型阶段迈向生产阶段，LangChain 也因此更加注重稳定性。
</Update>

<Update label="2024-02">
    **LangGraph 作为开源库发布**。

    最初的 LangChain 有两个重点：LLM 抽象和用于常见应用的高层接口。但缺少一个底层编排层，让开发者能精确控制 Agent 的执行流程。于是 LangGraph 应运而生。

    在构建 LangGraph 时，我们汲取了构建 LangChain 的经验教训，并添加了实践中发现必需的功能：流式输出、持久执行、短期记忆、人机协同等。
</Update>

<Update label="2024-10">
    LangGraph 成为构建任何超越单次 LLM 调用的 AI 应用的首选方式。

    随着开发者努力提升应用的可靠性，他们需要比高层接口更精细的控制。LangGraph 提供了这种底层灵活性。LangChain 中的大多数 Chain 和 Agent 被标记为已弃用，并提供了迁移到 LangGraph 的指南。不过，LangGraph 中保留了一个高层抽象：Agent 抽象。它构建在底层 LangGraph 之上，与 LangChain 的 ReAct Agent 具有相同的接口。
</Update>
