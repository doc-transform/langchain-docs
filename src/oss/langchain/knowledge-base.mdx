## 概述

本教程将帮助你熟悉 LangChain 的[文档加载器](/oss/langchain/retrieval#document-loaders)、[嵌入模型](/oss/langchain/retrieval#embedding-models)和[向量存储](/oss/langchain/retrieval#vector-store)抽象。这些抽象旨在支持从（向量）数据库和其他来源检索数据，以便与 LLM 工作流集成。它们对于需要在模型推理过程中获取数据进行推理的应用至关重要，例如检索增强生成，即 [RAG](/oss/langchain/retrieval)。

在本教程中，我们将基于一个 PDF 文档构建一个搜索引擎。这将允许我们根据输入查询检索 PDF 中的相似段落。本指南还包含一个基于搜索引擎的简化版 RAG 实现。

### 概念

本指南专注于文本数据的检索。我们将涵盖以下概念：

- [文档和文档加载器](/oss/integrations/document_loaders)
- [文本分割器](/oss/integrations/splitters)
- [嵌入模型](/oss/integrations/text_embedding)
- [向量存储](/oss/integrations/vectorstores)和[检索器](/oss/integrations/retrievers)

## 1. 文档和文档加载器

LangChain 实现了一个 @[Document] 抽象，用于表示一个文本单元及其关联元数据。它有三个属性：

:::python
- `page_content`：表示内容的字符串
- `metadata`：包含任意元数据的字典
- `id`：（可选）文档的字符串标识符
:::
:::js
- `pageContent`：表示内容的字符串
- `metadata`：包含任意元数据的字典
- `id`：（可选）文档的字符串标识符
:::

`metadata` 属性可以捕获文档来源、与其他文档的关系以及其他信息。注意，单个 @[`Document`] 对象通常表示较大文档的一个块。

我们可以根据需要生成示例文档：

:::python
```python

documents = [
    Document(
        page_content="Dogs are great companions, known for their loyalty and friendliness.",
        metadata={"source": "mammal-pets-doc"},
    ),
    Document(
        page_content="Cats are independent pets that often enjoy their own space.",
        metadata={"source": "mammal-pets-doc"},
    ),
]
```

const documents = [

### 文本分割

无论是信息检索还是下游的问答任务，以页为单位的粒度可能太粗。我们的最终目标是检索能够回答输入查询的 @[`Document`] 对象，对 PDF 进行更细粒度的分割有助于确保相关部分的语义不会被周围文本“淡化”。

我们可以使用[文本分割器](/oss/langchain/retrieval#text_splitters)来实现。这里我们将使用一个基于字符的简单文本分割器。我们将文档分割为 1000 字符的块，块之间有 200 字符的重叠。重叠有助于减少将某段叙述与其重要上下文分离的可能性。我们使用 `RecursiveCharacterTextSplitter`，它会使用常见分隔符（如换行符）递归分割文档，直到每个块达到合适的大小。这是通用文本场景的推荐文本分割器。

:::python
我们设置 `add_start_index=True`，以便将每个分割 Document 在原始 Document 中的起始字符索引保留为元数据属性 “start_index”。
```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000, chunk_overlap=200, add_start_index=True
)
all_splits = text_splitter.split_documents(docs)

print(len(all_splits))
```
:::
:::js
```typescript
import { RecursiveCharacterTextSplitter } from "@langchain/textsplitters";

const textSplitter = new RecursiveCharacterTextSplitter({
  chunkSize: 1000,
  chunkOverlap: 200,
});

const allSplits = await textSplitter.splitDocuments(docs);

console.log(allSplits.length);
```
:::

```text
514
```


## 2. 嵌入模型

向量搜索是存储和搜索非结构化数据（如非结构化文本）的常见方式。其思想是存储与文本关联的数值向量。给定一个查询，我们可以将其[嵌入](/oss/langchain/retrieval#embedding_models)为相同维度的向量，并使用向量相似度度量（如余弦相似度）来识别相关文本。

LangChain 支持[数十家提供商](/oss/integrations/text_embedding/)的嵌入模型。这些模型定义了如何将文本转换为数值向量。让我们选择一个模型：

:::python
<EmbeddingsTabsPy />

```python
vector_1 = embeddings.embed_query(all_splits[0].page_content)
vector_2 = embeddings.embed_query(all_splits[1].page_content)

assert len(vector_1) == len(vector_2)
print(f"Generated vectors of length {len(vector_1)}\n")
print(vector_1[:10])
```
:::
:::js
<EmbeddingsTabsJS />

```typescript
const vector1 = await embeddings.embedQuery(allSplits[0].pageContent);
const vector2 = await embeddings.embedQuery(allSplits[1].pageContent);

assert vector1.length === vector2.length;
console.log(`Generated vectors of length ${vector1.length}\n`);
console.log(vector1.slice(0, 10));
```

## 3. 向量存储

LangChain 的 @[VectorStore] 对象包含将文本和 @[`Document`] 对象添加到存储中的方法，以及使用各种相似度指标查询它们的方法。它们通常用[嵌入模型](/oss/langchain/retrieval#embedding_models)初始化，该模型决定如何将文本数据转换为数值向量。

LangChain 包含一套与不同向量存储技术的[集成](/oss/integrations/vectorstores)。某些向量存储由提供商托管（如各种云提供商），需要特定的凭据才能使用；某些（如 [Postgres](/oss/integrations/vectorstores/pgvector)）运行在独立的基础设施中，可以本地运行或通过第三方运行；其他的可以在内存中运行，适用于轻量级工作负载。让我们选择一个向量存储：

## 4. 检索器

LangChain 的 @[`VectorStore`] 对象不是 @[Runnable] 的子类。LangChain 的 @[Retriever] 是 Runnable，因此它们实现了一套标准方法（如同步和异步的 `invoke` 和 `batch` 操作）。虽然我们可以从向量存储构建检索器，但检索器也可以对接非向量存储的数据源（如外部 API）。

:::python
我们可以自己实现一个简单版本，无需继承 `Retriever`。如果我们确定了用哪个方法检索文档，就可以轻松创建一个 Runnable。下面我们基于 `similarity_search` 方法构建一个：


```python
from typing import List

from langchain_core.documents import Document
from langchain_core.runnables import chain


@chain
def retriever(query: str) -> List[Document]:
    return vector_store.similarity_search(query, k=1)


retriever.batch(
    [
        "How many distribution centers does Nike have in the US?",
        "When was Nike incorporated?",
    ],
)
```

retriever = vector_store.as_retriever(

)

)
