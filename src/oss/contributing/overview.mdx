## LLM 的合理使用

生成式 AI 是贡献者的有力工具，但与任何工具一样，需要结合批判性思维和良好的判断力来使用。

我们鼓励贡献者在有助于提升效率时合理使用 AI 工具。但是，AI 辅助必须伴随有意义的人工介入、判断和上下文理解。**如果创建一个 Pull Request 所需的人工投入少于维护者审核它所需的精力，那么该贡献不应提交。**

当贡献者的全部工作（代码修改、文档更新、PR 描述）都由 LLM 生成时，我们会感到困扰。这类走马观花式的贡献虽然出发点是好的，在上下文相关性、准确性和质量方面往往达不到要求。大量的自动化贡献实质上是对我们人工审核精力的消耗。

**我们将关闭那些明显属于低质量、AI 生成的垃圾 Pull Request 和 Issue。**

工具越强大，责任越重大。
