**集成是 LangChain 的核心组成部分。**

LangChain 为构建 LLM 应用所需的多种关键组件（语言模型、向量存储等）提供了标准接口。贡献一个集成有助于扩展 LangChain 生态系统，并让你的服务被数百万开发者发现。

## 为什么要为 LangChain 贡献集成？

<Card title="可发现性" icon="search">
    LangChain 是最广泛使用的 LLM 应用开发框架，月下载量超过 2000 万。
</Card>

<Card title="互操作性" icon="refresh">
    LangChain 组件暴露标准接口，使开发者可以轻松地在不同组件之间切换。如果你实现了 LangChain 集成，任何使用其他同类组件的开发者都可以方便地替换为你的实现。
</Card>

<Card title="最佳实践" icon="star">
    通过标准接口，LangChain 组件鼓励并促进最佳实践（流式传输、异步处理等），从而提升开发体验和应用性能。
</Card>

## 适合集成的组件

虽然任何组件都可以集成到 LangChain 中，但以下几类集成是我们重点鼓励的：

**推荐集成 ✅**：
- [**聊天模型**](/oss/integrations/chat)：最活跃使用的组件类型
- [**工具 / 工具集**](/oss/integrations/tools)：为 Agent 提供能力支持
- [**检索器**](/oss/integrations/retrievers)：RAG 应用的核心组件
- [**嵌入模型**](/oss/integrations/text_embedding)：向量操作的基础
- [**向量存储**](/oss/integrations/vectorstores)：语义搜索的关键设施
- [**沙箱**](/oss/deepagents/sandboxes)：为 Deep Agents 提供安全的代码运行环境

<Accordion title="第三方沙箱集成的额外准入标准">
请注意，我们仅在以下条件下收录第三方沙箱集成：

- 该集成由提供沙箱服务的公司编写并维护。
- **或者**该集成被广泛使用，即在 PyPI 或 npm 上的日下载量不低于 10,000 次。
</Accordion>

**不推荐集成 ❌**：
- **LLM（文本补全模型）**：已被 [聊天模型](/oss/integrations/chat) 取代
- [**文档加载器**](/oss/integrations/document_loaders)：维护成本较高
- [**键值存储**](/oss/integrations/stores)：使用量有限
- **文档转换器**：应用场景较窄
- **模型缓存**：属于基础设施层面
- **图结构**：抽象复杂度较高
- **消息历史**：存储层抽象
- **回调**：系统级组件
- **聊天加载器**：需求有限
- **适配器**：边缘用例

## 如何贡献集成

<Steps>
    <Step title="确认资格">
        确认你的集成属于我们当前接受的 [推荐组件](#适合集成的组件) 类型。
    </Step>

    <Step title="实现你的包">
        <Card title="如何实现 LangChain 集成" icon="link" href="/oss/contributing/implement-langchain" arrow />
    </Step>

    <Step title="通过标准测试">
        如适用，为你的集成实现 LangChain [标准测试](/oss/contributing/standard-tests-langchain) 套件并确保测试通过。
    </Step>

    <Step title="发布集成">
        <Card title="如何发布集成" icon="upload" href="/oss/contributing/publish-langchain" arrow />
    </Step>

    <Step title="添加文档">
        提交 PR 将你的集成文档添加到 LangChain 官方文档中。

        <Accordion title="集成文档编写指南" icon="book">
            一个集成的价值取决于其文档质量。为了确保用户体验的一致性，所有新集成都必须提供文档。我们为每种集成类型提供了标准模板，供你复制和修改。

            在 LangChain [文档仓库](https://github.com/langchain-ai/docs) 中提交新的 PR，在 `src/oss/python/integrations/<component_type>/integration_name.mdx` 对应目录下创建新文件，使用相应的模板文件：

            - [聊天模型](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/chat/TEMPLATE.mdx)
            - [工具和工具集](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/tools/TEMPLATE.mdx)
            - [检索器](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/retrievers/TEMPLATE.mdx)
            - 文本分割器 - 即将推出
            - 嵌入模型 - 即将推出
            - [向量存储](https://github.com/langchain-ai/docs/blob/main/src/oss/python/integrations/vectorstores/TEMPLATE.mdx)
            - 文档加载器 - 即将推出
            - 键值存储 - 即将推出

            关于 API 参考文档，请在仓库中提交 issue，由维护者添加。
        </Accordion>
    </Step>

    <Step title="联合推广" icon="speakerphone">
        （可选）与 LangChain 团队合作进行 [联合推广](/oss/contributing/comarketing)。
    </Step>
</Steps>
