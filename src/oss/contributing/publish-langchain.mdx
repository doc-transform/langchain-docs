**让你的集成面向社区可用。**

:::python
<Warning>
**请勿向 LangChain 或 Deep Agents 仓库提交集成 PR。**

新的集成应作为**独立的 PyPI 包**发布在你自己的 GitHub 组织或个人账户下（例如 `langchain-yourservice`），而不是作为 PR 提交到 [`langchain-ai/langchain`](https://github.com/langchain-ai/langchain) 仓库。

主仓库仅包含由 LangChain 团队维护的少量第一方集成（如 OpenAI、Anthropic 和 Ollama）。
</Warning>

你的包已经完成实现和测试，现在可以发布并添加文档，让社区能够发现和使用它。

## 发布你的包

<Info>
    本指南假设你已完成包的实现和测试。如果尚未完成，请参阅 [实现指南](/oss/contributing/implement-langchain) 和 [测试指南](/oss/contributing/standard-tests-langchain)。
</Info>

本指南以 PyPI 作为包注册中心。你也可以选择发布到其他注册中心，具体步骤可能会有所不同。

### 设置凭据

首先，确保你拥有 PyPI 账户：

<AccordionGroup>
    <Accordion title="如何创建 PyPI Token" icon="key">
        <Steps>
            <Step title="创建账户">
                前往 [PyPI 网站](https://pypi.org/) 注册账户
            </Step>

            <Step title="验证邮箱">
                点击 PyPI 发送的验证链接完成邮箱验证
            </Step>

            <Step title="启用双因素认证">
                前往账户设置，点击"Generate Recovery Codes"以启用双因素认证。要生成 API Token，**必须**先启用双因素认证
            </Step>

            <Step title="生成 Token">
                前往账户设置，[生成新的 API Token](https://pypi.org/manage/account/token/)
            </Step>
        </Steps>
    </Accordion>
</AccordionGroup>

## 添加文档

要将你的包文档添加到本站的 [集成标签页](/oss/integrations/providers/overview) 下，你需要创建相关文档页面并向 [LangChain 文档仓库](https://github.com/langchain-ai/docs) 提交 PR。

### 编写文档

根据你构建的集成类型，你需要创建不同类型的文档页面。LangChain 为各类集成提供了模板，帮助你快速上手。

<Tip>
    要参考现有文档，可以查看 [集成列表](/oss/integrations/providers/overview) 找到与你类似的集成。

    要查看某个文档页面的原始 Markdown 格式，点击页面右上角"Copy page"旁边的下拉按钮，选择"View as Markdown"。
</Tip>

### 向文档仓库提交 PR

Fork [LangChain 文档仓库](https://github.com/langchain-ai/docs)（不是主 `langchain` 仓库）到你的个人 GitHub 账户下，并克隆到本地。创建一个新分支用于你的集成。复制模板并使用你喜欢的 Markdown 编辑器进行修改。编写文档时请务必参考并遵循 [文档编写指南](/oss/contributing/documentation)。

<Info>
    此 PR 仅用于**文档**。你的集成包本身应位于你自己的 GitHub 组织或个人账户下的独立仓库中，并作为独立包发布到 PyPI。
</Info>

<Warning>
    以下情况我们可能会拒绝 PR 或要求修改：
    - CI 检查未通过
    - 存在严重的语法错误或拼写错误
    - [Mintlify 组件](/oss/contributing/documentation#mintlify-components) 使用不正确
    - 页面缺少 [frontmatter](/oss/contributing/documentation#page-structure)
    - 缺少 [本地化](/oss/contributing/documentation#localization) 内容（如适用）
    - [代码示例](/oss/contributing/documentation#in-code-documentation) 无法运行或有错误
    - 不符合 [质量标准](/oss/contributing/documentation#quality-standards)
</Warning>

由于我们需要处理大量 PR，请耐心等待。我们会尽快审核你的 PR 并提供反馈或合并。**请勿反复 @ 维护者催促你的 PR。**
