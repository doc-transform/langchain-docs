并非所有 Agent 交互都是聊天。有时 Agent 正在执行多步计划，展示进度的最佳方式是一个实时更新的**待办列表**。Deep Agent 待办列表模式直接从 Agent 状态中读取 `todos` 数组，随着 Agent 执行计划渲染每个项目及其当前状态。它是构建在同一个 `useStream` 钩子之上的进度仪表板。它表明 Agent 状态可以驱动任何 UI，而不仅仅是消息气泡。

## 工作原理

在 LangGraph Agent 中，状态不仅限于消息。你可以定义**自定义状态键**来保存任意数据。在本例中，就是一个 `todos` 数组。当 Agent 执行计划时，它将每个待办项的状态从 `"pending"` 更新为 `"in_progress"` 再到 `"completed"`。`useStream` 钩子通过 `stream.values` 暴露这些自定义状态值，你的 UI 会响应式地渲染它们。

流程如下：

1. 用户提交请求
2. Agent 创建计划并在其状态中填充 `todos`
3. Agent 开始执行每个待办项，依次经历 `pending` → `in_progress` → `completed`
4. `stream.values.todos` 随着 Agent 进度实时更新
5. 你的 UI 用当前状态重新渲染待办列表

## 设置 useStream

不需要特殊配置。将 `useStream` 指向你的 Agent，从 `stream.values` 读取 `todos`。

:::python

定义一个与 Agent 状态 schema 匹配的 TypeScript 接口，并将其作为类型参数传递给 `useStream`，以获得类型安全的状态值访问（包括 `todos` 等自定义状态键）。在下面的示例中，将 `typeof myAgent` 替换为你的接口名称：

```ts
import type { BaseMessage } from "@langchain/core/messages";

interface TodoItem {
  title: string;
  status: "pending" | "in_progress" | "completed";
  description?: string;
}

interface AgentState {
  messages: BaseMessage[];
  todos: TodoItem[];
}
```


## Todo 接口

数组中每个待办项都有简单的结构：

```ts
interface Todo {
  status: "pending" | "in_progress" | "completed";
  content: string;
}
```

| 属性 | 描述 |
| --- | --- |
| `status` | 此任务的当前状态。可选值：`pending`（未开始）、`in_progress`（Agent 正在处理）、`completed`（已完成） |
| `content` | 任务内容的可读描述 |

Agent 在创建计划时填充此数组，然后在执行每个步骤时更新各个项目。

## 构建 TodoList 组件

待办列表使用状态图标、颜色编码和视觉样式渲染每个项目，反映其当前状态：

```tsx
function TodoList({ todos }: { todos: Todo[] }) {
  const completed = todos.filter((t) => t.status === "completed").length;
  const percentage = todos.length
    ? Math.round((completed / todos.length) * 100)
    : 0;

  return (
    <div className="rounded-lg border bg-white p-4 shadow-sm">
      <div className="mb-4 flex items-center justify-between">
        <h2 className="text-lg font-semibold">Agent Progress</h2>
        <span className="text-sm text-gray-500">
          {completed}/{todos.length} tasks
        </span>
      </div>

      <ProgressBar percentage={percentage} />

      <ul className="mt-4 space-y-2">
        {todos.map((todo, i) => (
          <TodoItem key={i} todo={todo} />
        ))}
      </ul>
    </div>
  );
}
```

## 单个待办项

每个项目都有状态图标、颜色编码文本，已完成的任务带有删除线样式：

```tsx
function TodoItem({ todo }: { todo: Todo }) {
  const config = {
    pending: {
      icon: "○",
      textClass: "text-gray-600",
      bgClass: "bg-gray-50",
      iconClass: "text-gray-400",
    },
    in_progress: {
      icon: "◉",
      textClass: "text-amber-800",
      bgClass: "bg-amber-50 border-amber-200",
      iconClass: "text-amber-500 animate-pulse",
    },
    completed: {
      icon: "✓",
      textClass: "text-green-800 line-through",
      bgClass: "bg-green-50 border-green-200",
      iconClass: "text-green-500",
    },
  };

  const style = config[todo.status];

  return (
    <li
      className={`flex items-start gap-3 rounded-md border px-3 py-2 ${style.bgClass}`}
    >
      <span className={`mt-0.5 text-lg leading-none ${style.iconClass}`}>
        {style.icon}
      </span>
      <span className={`text-sm ${style.textClass}`}>{todo.content}</span>
    </li>
  );
}
```

`in_progress` 图标使用 `animate-pulse` 来引起用户对当前活跃任务的注意。

## 计算进度

直接从 todos 数组推导进度指标：

```ts
const todos = stream.values?.todos ?? [];

const completed = todos.filter((t) => t.status === "completed").length;
const inProgress = todos.filter((t) => t.status === "in_progress").length;
const pending = todos.filter((t) => t.status === "pending").length;
const percentage = todos.length
  ? Math.round((completed / todos.length) * 100)
  : 0;
```

这些值随着 Agent 修改其状态而响应式更新，保持进度条和计数器同步。

## 与聊天消息结合

待办列表与常规聊天界面配合使用。实用的布局是将待办列表作为持久的侧边栏或头部面板展示，聊天消息在下方：

```tsx
function TodoAgentLayout() {
  const stream = useStream<typeof myAgent>({
    apiUrl: AGENT_URL,
    assistantId: "deep_agent_todo_list",
  });

  const todos = stream.values?.todos ?? [];

  return (
    <div className="flex h-screen flex-col">
      {todos.length > 0 && (
        <div className="border-b bg-gray-50 p-4">
          <TodoList todos={todos} />
        </div>
      )}

      <main className="flex-1 overflow-y-auto p-6">
        <div className="mx-auto max-w-2xl space-y-4">
          {stream.messages.map((msg) => (
            <Message key={msg.id} message={msg} />
          ))}
        </div>
      </main>

      <ChatInput
        onSubmit={(text) =>
          stream.submit({ messages: [{ type: "human", content: text }] })
        }
        isLoading={stream.isLoading}
      />
    </div>
  );
}
```

<Tip>
仅在 `todos.length > 0` 时展示待办列表。在 Agent 创建计划之前没有内容可展示，显示空组件会浪费空间。
</Tip>

## 超越 todos 的自定义状态

这个模式展示了一个强大的原则：`stream.values` 可以暴露 Agent 定义的**任何自定义状态**，不仅限于消息。`todos` 数组只是一个例子。你可以用相同的方法实现：

- **进度指标**：`stream.values.progress` 包含数字完成数据
- **生成的产物**：`stream.values.document` 包含 Agent 正在构建的结构化文档
- **决策日志**：`stream.values.decisions` 跟踪 Agent 做出的每个选择
- **资源列表**：`stream.values.sources` 包含 Agent 找到的链接和引用

```ts
// Any custom state key your agent defines is accessible
const document = stream.values?.document;
const sources = stream.values?.sources ?? [];
const confidence = stream.values?.confidence_score;
```

<Info>
自定义状态键在 LangGraph 图的状态 schema 中定义。`useStream` 钩子会自动将它们包含在 `stream.values` 中，无需额外的客户端配置。
</Info>

## 动画过渡

待办项状态变化是实时发生的，流畅的动画使这些变化感觉精致而非突兀：

```tsx
function TodoItem({ todo }: { todo: Todo }) {
  return (
    <li
      className={`
        flex items-start gap-3 rounded-md border px-3 py-2
        transition-all duration-300 ease-in-out
        ${getStatusStyles(todo.status)}
      `}
    >
      <span
        className={`
          mt-0.5 text-lg leading-none transition-colors duration-300
          ${getIconStyles(todo.status)}
        `}
      >
        {getStatusIcon(todo.status)}
      </span>
      <span
        className={`
          text-sm transition-all duration-300
          ${todo.status === "completed" ? "line-through opacity-60" : ""}
        `}
      >
        {todo.content}
      </span>
    </li>
  );
}
```

`transition-all duration-300` 类确保颜色变化、删除线和透明度变化都能平滑过渡。

## 使用场景

待办列表模式适用于 Agent 执行结构化计划的任何场景：

- **项目规划**：Agent 将项目拆分为任务并依次执行
- **研究工作流**：每个研究问题成为一个待办项，Agent 逐一调查并完成
- **数据处理**：采集、验证、转换和导出等步骤各自成为一个待办项
- **引导流程**：Agent 逐步完成设置步骤，配置服务时逐一打勾
- **报告生成**：报告的各个章节成为待办项：收集数据、分析趋势、撰写摘要、格式化输出

## 处理空状态和加载状态

处理 Agent 创建计划之前的初始状态：

```tsx
function TodoList({ todos, isLoading }: { todos: Todo[]; isLoading: boolean }) {

    return null;

    <div className="rounded-lg border bg-white p-4 shadow-sm">

    </div>
  );
}
```

## 最佳实践

- **突出展示待办列表**。它是基于计划的 Agent 的主要进度指示器，不要把它隐藏在下方。
- **为状态转换添加动画**。流畅的过渡让 Agent 感觉更灵敏。在背景色、文本装饰和透明度上使用 CSS 过渡。
- **只高亮一个 `in_progress` 项**。Agent 通常一次只处理一个任务。如果多个项目显示为 `in_progress`，UI 会变得嘈杂。考虑只对第一个使用脉冲效果。
- **折叠或淡化已完成项**。随着列表增长，已完成的项目变得不那么重要。降低它们的视觉权重，让用户关注仍在进行的内容。
- **展示完成百分比**。一个简单的数字如"67% 已完成"即使在远处也能立即理解。
- **保持待办列表同步**。因为 `stream.values` 响应式更新，待办列表会自动保持最新。不要添加手动轮询或刷新逻辑。
