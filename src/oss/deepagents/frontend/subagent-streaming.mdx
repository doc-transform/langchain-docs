当协调者 Agent 生成专业子 Agent（研究员、分析师、撰稿人）时，你需要将协调者的消息与每个子 Agent 的流式输出分开渲染。在 `useStream` 中设置 `filterSubagentMessages: true` 来清晰地分离这两个流，然后使用 `getSubagentsByMessage` 将每个子 Agent 的进度卡片关联到触发它的协调者消息。

## 为什么要过滤子 Agent 消息

如果不过滤，每个子 Agent 产生的每个 token 都会交错出现在协调者的消息流中，导致不可读。设置 `filterSubagentMessages: true` 后：

- `stream.messages` 仅包含协调者的消息
- 每个子 Agent 的内容可通过 `stream.subagents` 和 `stream.getSubagentsByMessage` 访问
- UI 保持整洁：协调者的推理与专家的工作分开展示

这种分离让你可以在一处渲染协调者的消息，并在触发子 Agent 的协调者消息下方精确地附加每个子 Agent 的进度卡片。

## 设置 useStream

始终设置 `filterSubagentMessages: true`。这会从主消息流中移除子 Agent 的 token，以便你可以独立渲染协调者消息和子 Agent 输出。

## 提交消息时启用子图流式传输

提交消息时，启用子图流式传输并设置适当的递归限制。Deep Agent 工作流通常涉及多层嵌套子图，因此更高的递归限制可以防止过早终止：

```ts
stream.submit(
  { messages: [{ type: "human", content: text }] },
  { streamSubgraphs: true }
);
```

<Note>
DeepAgents 默认设置的递归限制为 10,000，对大多数多专家设置来说足够了。如有需要，你可以通过 `config.recursion_limit` 覆盖此值。
</Note>

## SubagentStreamInterface 接口

每个子 Agent 暴露一个 `SubagentStreamInterface`，包含子 Agent 任务、状态和时间的元数据：

```ts
interface SubagentStreamInterface {
  id: string;
  status: "pending" | "running" | "complete" | "error";
  messages: BaseMessage[];
  result: string | undefined;
  toolCall: {
    id: string;
    name: string;
    args: {
      description: string;
      subagent_type: string;
      [key: string]: unknown;
    };
  };
  startedAt: number | undefined;
  completedAt: number | undefined;
}
```

| 属性 | 描述 |
| --- | --- |
| `id` | 此子 Agent 实例的唯一标识符 |
| `status` | 生命周期状态：`pending` → `running` → `complete` 或 `error` |
| `messages` | 子 Agent 自己的消息流，实时更新 |
| `result` | 最终输出文本，仅在 `status` 为 `"complete"` 时可用 |
| `toolCall` | 生成此子 Agent 的工具调用，包含任务元数据 |
| `toolCall.args.description` | 协调者分配给此子 Agent 的任务描述 |
| `toolCall.args.subagent_type` | 专家的类型或名称（例如 `"researcher"`、`"analyst"`） |
| `startedAt` | 子 Agent 开始执行的时间戳 |
| `completedAt` | 子 Agent 完成的时间戳 |

## 将子 Agent 关联到消息

`getSubagentsByMessage` 方法返回由特定 AI 消息生成的子 Agent。这让你可以直接在触发它们的协调者消息下方渲染子 Agent 卡片：

```ts
const turnSubagents = stream.getSubagentsByMessage(msg.id);
```

这返回一个 `SubagentStreamInterface` 对象数组。如果该消息没有生成任何子 Agent，则返回空数组。

## 构建 SubagentCard 组件

每个子 Agent 卡片展示专家名称、任务描述、流式内容或最终结果以及时间信息：

```tsx
import { AIMessage } from "@langchain/core/messages";

function SubagentCard({
  subagent,
}: {
  subagent: SubagentStreamInterface;
}) {
  const [expanded, setExpanded] = useState(true);

  const title =
    subagent.toolCall?.args?.subagent_type ?? `Agent ${subagent.id}`;
  const description = subagent.toolCall?.args?.description ?? "";

  const lastAIMessage = subagent.messages
    .filter(AIMessage.isInstance)
    .at(-1);

  const displayContent =
    subagent.status === "complete"
      ? subagent.result
      : typeof lastAIMessage?.content === "string"
        ? lastAIMessage.content
        : "";

  const elapsed = getElapsedTime(subagent.startedAt, subagent.completedAt);

  return (
    <div className="rounded-lg border bg-white shadow-sm">
      <button
        onClick={() => setExpanded(!expanded)}
        className="flex w-full items-center justify-between p-4"
      >
        <div className="flex items-center gap-3">
          <StatusIcon status={subagent.status} />
          <div>
            <h4 className="font-semibold capitalize">{title}</h4>
            <p className="text-xs text-gray-500">{description}</p>
          </div>
        </div>
        <div className="flex items-center gap-2">
          {elapsed && (
            <span className="text-xs text-gray-400">{elapsed}</span>
          )}
          <StatusBadge status={subagent.status} />
        </div>
      </button>

      {expanded && displayContent && (
        <div className="border-t px-4 py-3">
          <div className="prose prose-sm max-w-none line-clamp-6">
            {displayContent}
            {subagent.status === "running" && (
              <span className="inline-block h-4 w-1 animate-pulse bg-blue-500" />
            )}
          </div>
        </div>
      )}
    </div>
  );
}

function getElapsedTime(
  startedAt: number | undefined,
  completedAt: number | undefined
): string | null {
  if (!startedAt) return null;
  const end = completedAt ?? Date.now();
  const seconds = Math.round((end - startedAt) / 1000);
  if (seconds < 60) return `${seconds}s`;
  return `${Math.floor(seconds / 60)}m ${seconds % 60}s`;
}
```

## 状态图标和徽章

一致的视觉指示器帮助用户快速识别子 Agent 状态：

```tsx
function StatusIcon({ status }: { status: SubagentStreamInterface["status"] }) {
  switch (status) {
    case "pending":
      return <span className="text-gray-400">○</span>;
    case "running":
      return <span className="animate-spin text-blue-500">◉</span>;
    case "complete":
      return <span className="text-green-500">✓</span>;
    case "error":
      return <span className="text-red-500">✕</span>;
  }
}

function StatusBadge({ status }: { status: SubagentStreamInterface["status"] }) {
  const styles = {
    pending: "bg-gray-100 text-gray-600",
    running: "bg-blue-100 text-blue-700",
    complete: "bg-green-100 text-green-700",
    error: "bg-red-100 text-red-700",
  };

  return (
    <span className={`rounded-full px-2 py-0.5 text-xs font-medium ${styles[status]}`}>
      {status}
    </span>
  );
}
```

## 进度跟踪

展示进度条和计数器，让用户知道有多少子 Agent 已完成：

```tsx
function SubagentProgress({
  subagents,
}: {
  subagents: SubagentStreamInterface[];
}) {
  const completed = subagents.filter((s) => s.status === "complete").length;
  const total = subagents.length;
  const percentage = total > 0 ? Math.round((completed / total) * 100) : 0;

  return (
    <div className="space-y-1">
      <div className="flex items-center justify-between text-xs text-gray-500">
        <span>Subagent progress</span>
        <span>
          {completed}/{total} complete
        </span>
      </div>
      <div className="h-2 overflow-hidden rounded-full bg-gray-200">
        <div
          className="h-full rounded-full bg-blue-500 transition-all duration-300"
          style={{ width: `${percentage}%` }}
        />
      </div>
    </div>
  );
}
```

## 渲染消息与子 Agent 卡片

关键布局模式是渲染每个协调者消息，如果该消息生成了子 Agent，则紧接其下方渲染子 Agent 卡片：

```tsx
function MessageWithSubagents({
  message,
  subagents,
}: {
  message: BaseMessage;
  subagents: SubagentStreamInterface[];
}) {
  if (message.type === "human") {
    return <HumanMessage content={message.content} />;
  }

  return (
    <div className="space-y-3">
      {message.content && (
        <div className="prose prose-sm max-w-none">
          {message.content}
        </div>
      )}

      {subagents.length > 0 && (
        <div className="ml-4 space-y-3 border-l-2 border-blue-200 pl-4">
          <SubagentProgress subagents={subagents} />
          {subagents.map((subagent) => (
            <SubagentCard key={subagent.id} subagent={subagent} />
          ))}
        </div>
      )}
    </div>
  );
}
```

## 合成指示器

所有子 Agent 完成后，协调者需要时间将它们的结果合成为最终响应。在此阶段展示一个清晰的指示器：

```tsx
function SynthesisIndicator({
  subagents,
  isLoading,
}: {
  subagents: SubagentStreamInterface[];
  isLoading: boolean;
}) {
  const allComplete =
    subagents.length > 0 &&
    subagents.every((s) => s.status === "complete" || s.status === "error");

  if (!allComplete || !isLoading) return null;

  return (
    <div className="flex items-center gap-2 rounded-lg bg-purple-50 px-4 py-2 text-sm text-purple-700">
      <span className="animate-spin">⟳</span>
      Synthesizing results from {subagents.length} subagent
      {subagents.length !== 1 ? "s" : ""}...
    </div>
  );
}
```

<Tip>
对于复杂的多专家工作流，合成阶段可能需要几秒钟。清晰的"正在合成结果..."指示器可以防止用户认为 Agent 卡住了。
</Tip>

## 调试未过滤的输出

在开发过程中，你可以临时设置 `filterSubagentMessages: false` 来查看主消息流中所有子 Agent 的原始交错输出。这对验证子 Agent token 是否正确流动很有用，但不应在生产 UI 中使用。

## 使用场景

当 Agent 工作流涉及以下场景时，Deep Agent 子 Agent 卡片是正确的选择：

- **深度研究**：协调者派遣研究人员调查问题的不同方面，然后综合他们的发现
- **多专家分析**：领域专家（法律、金融、技术）各自贡献其视角
- **复杂任务分解**：规划者将大任务拆分为子任务，并将每个子任务分配给专家工作者
- **代码审查流水线**：独立的 Agent 分别处理安全审查、代码风格检查、性能分析和文档审查

## 访问完整的子 Agent 映射

除了按消息查找，你还可以通过 `stream.subagents` 一次访问所有子 Agent：

```ts
const allSubagents = [...stream.subagents.values()];
const running = allSubagents.filter((s) => s.status === "running");
const completed = allSubagents.filter((s) => s.status === "complete");
const errors = allSubagents.filter((s) => s.status === "error");
```

这对构建全局进度指示器或仪表板很有用，可以汇总所有子 Agent 活动，不管它们是由哪条协调者消息生成的。

## 最佳实践

- **始终设置 `filterSubagentMessages: true`**。未过滤的流会产生协调者和子 Agent token 的不可读交错。
- **展示任务描述**。`toolCall.args.description` 字段告诉用户每个子 Agent 被要求做什么。始终突出展示。
- **使用可折叠卡片**。在有 5 个以上子 Agent 的工作流中，自动折叠已完成的卡片，让用户专注于活跃的工作。
- **展示时间数据**。展示每个子 Agent 花费的时间帮助用户了解性能特征并识别瓶颈。
- **设置适当的递归限制**。具有嵌套子图的 Deep Agent 工作流需要比默认值 25 更高的限制。建议从 100 开始。
- **按子 Agent 处理错误**。一个子 Agent 失败不应导致整个 UI 崩溃。在该子 Agent 的卡片中展示错误，同时其他子 Agent 继续运行。
