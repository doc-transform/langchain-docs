Deep Agents 是开始构建 LLM 驱动的 Agent 和应用的最简单方式——内置任务规划、用于上下文管理的文件系统、子 Agent 生成和长期记忆等能力。
你可以将 Deep Agents 用于任何任务，包括复杂的多步骤任务。

我们将 `deepagents` 视为 ["Agent 能力框架"](/oss/concepts/products#agent-harnesses-like-the-deep-agents-sdk)。它与其他 Agent 框架拥有相同的核心工具调用循环，但具备内置的工具和能力。

:::python
[`deepagents`](https://pypi.org/project/deepagents/) 是一个基于 [LangChain](/oss/langchain/) Agent 核心构建块的独立库。它使用 [LangGraph](/oss/langgraph/) 运行时提供持久化执行、流式输出、人工干预等功能。
:::

:::js
[`deepagents`](https://www.npmjs.com/package/deepagents) 是一个基于 [LangChain](/oss/langchain/) Agent 核心构建块的独立库，使用 [LangGraph](/oss/langgraph/) 的工具在生产环境中运行 Agent。
:::

`deepagents` 库包含：
- **Deep Agents SDK**：一个用于构建能处理任何任务的 Agent 的包
- **Deep Agents CLI**：一个基于 `deepagents` 包构建的终端编码 Agent

[LangChain](/oss/langchain/) 是为你的 Agent 提供核心构建块的框架。
要了解 LangChain、LangGraph 和 Deep Agents 之间的区别，请参阅[框架、运行时和能力框架](/oss/concepts/products)。

## <Icon icon="wand" /> 创建 Deep Agent

:::python
```python
# pip install -qU deepagents
from deepagents import create_deep_agent

def get_weather(city: str) -> str:
    """Get weather for a given city."""
    return f"It's always sunny in {city}!"

agent = create_deep_agent(
    tools=[get_weather],
    system_prompt="You are a helpful assistant",
)

# Run the agent
agent.invoke(
    {"messages": [{"role": "user", "content": "what is the weather in sf"}]}
)
```
:::

:::js
```ts
import * as z from "zod";
// npm install deepagents langchain @langchain/core
import { createDeepAgent } from "deepagents";
import { tool } from "langchain";

const getWeather = tool(
  ({ city }) => `It's always sunny in ${city}!`,
  {
    name: "get_weather",
    description: "Get the weather for a given city",
    schema: z.object({
      city: z.string(),
    }),
  },
);

const agent = createDeepAgent({
  tools: [getWeather],
  system: "You are a helpful assistant",
});

console.log(
  await agent.invoke({
    messages: [{ role: "user", content: "What's the weather in Tokyo?" }],
  })
);
```
:::

参阅[快速开始](/oss/deepagents/quickstart/)和[自定义指南](/oss/deepagents/customization/)，开始使用 Deep Agents 构建你自己的 Agent 和应用。

<Tip>
使用 [LangSmith](/langsmith/home) 追踪请求、调试 Agent 行为并评估输出。设置 `LANGSMITH_TRACING=true` 和 API 密钥即可开始。
</Tip>

## 何时使用 Deep Agents

当你想构建以下能力的 Agent 时，使用 **Deep Agents SDK**：
- **处理复杂的多步骤任务** — 需要规划和分解的任务
- **管理大量上下文** — 通过文件系统工具
- **切换文件系统后端** — 使用内存状态、本地磁盘、持久化存储、[沙箱](/oss/deepagents/sandboxes)或[自定义后端](/oss/deepagents/backends)
- **委派工作** — 给专业子 Agent 以实现上下文隔离
- **持久化记忆** — 跨对话和线程保存记忆

:::python
如果要构建更简单的 Agent，可以考虑使用 LangChain 的 [`create_agent`](/oss/langchain/agents) 或构建自定义的 [LangGraph](/oss/langgraph/overview) 工作流。
:::

:::js
如果要构建更简单的 Agent，可以考虑使用 LangChain 的 [`createAgent`](/oss/langchain/agents) 或构建自定义的 [LangGraph](/oss/langgraph/overview) 工作流。
:::

当你想在命令行中使用交互式 Deep Agent 进行编码或其他任务时，使用 **Deep Agents CLI**：

- **自定义** Agent 的技能和记忆。
- **教导** Agent 学习你的偏好、常用模式和自定义项目知识。
- **执行代码** — 在本地机器或沙箱中。

## 核心能力

<Card title="规划与任务分解" icon="timeline">
    Deep Agents 内置了 [`write_todos`](/oss/langchain/middleware/built-in#to-do-list) 工具，使 Agent 能够将复杂任务分解为离散步骤、跟踪进度并在出现新信息时调整计划。
</Card>
<Card title="上下文管理" icon="scissors">
    文件系统工具（[`ls`](/oss/deepagents/harness#file-system-access)、[`read_file`](/oss/deepagents/harness#file-system-access)、[`write_file`](/oss/deepagents/harness#file-system-access)、[`edit_file`](/oss/deepagents/harness#file-system-access)）允许 Agent 将大量上下文卸载到内存或文件系统存储，避免上下文窗口溢出，并支持处理可变长度的工具结果。
</Card>
<Card title="可插拔文件系统后端" icon="plug">
    虚拟文件系统由[可插拔后端](/oss/deepagents/backends)驱动，你可以根据使用场景进行切换。可选择内存状态、本地磁盘、LangGraph 存储以实现跨线程持久化、[沙箱](/oss/deepagents/sandboxes)以实现隔离代码执行（Modal、Daytona、Deno），或通过组合路由混合使用多个后端。你还可以实现自定义后端。
</Card>
<Card title="子 Agent 生成" icon="users-group">
    内置的 `task` 工具使 Agent 能够生成专业子 Agent 以实现上下文隔离。这能保持主 Agent 的上下文简洁，同时仍能深入处理特定子任务。
</Card>
<Card title="长期记忆" icon="database">
    使用 LangGraph 的[记忆存储](/oss/langgraph/persistence#memory-store)扩展 Agent 的跨线程持久化记忆。Agent 可以保存和检索先前对话中的信息。
</Card>
