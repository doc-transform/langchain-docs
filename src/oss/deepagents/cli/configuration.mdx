CLI 将其配置存储在 `~/.deepagents/` 目录中。主要配置文件包括：

| 文件 | 格式 | 用途 |
|------|--------|---------|
| `config.toml` | [TOML](https://toml.io/) | 模型默认值、提供商设置、构造函数参数、配置文件覆盖、MCP 信任存储 |
| `hooks.json` | JSON | 订阅 CLI 生命周期事件的外部工具 |
| `.mcp.json` | JSON | MCP 服务器定义（也可从项目目录自动发现） |

## 配置文件

`~/.deepagents/config.toml` 允许你自定义模型提供商、设置默认值，以及向模型构造函数传递额外参数。

### 默认模型和最近使用模型

```toml
[models]
default = "ollama:qwen3:4b"             # 你有意设置的长期偏好
recent = "anthropic:claude-sonnet-4-5"   # 上次 /model 切换时自动写入
```

`[models].default` 始终优先于 `[models].recent`。`/model` 命令只会写入 `[models].recent`，因此你配置的默认值永远不会被会话中途的切换所覆盖。要移除默认值，请使用 `/model --default --clear` 或直接从配置文件中删除 `default` 键。

### 提供商配置

每个提供商是 `[models.providers]` 下的一个 TOML 表：

```toml
[models.providers.<name>]
models = ["gpt-4o"]
api_key_env = "OPENAI_API_KEY"
base_url = "https://api.openai.com/v1"
class_path = "my_package.models:MyChatModel"

[models.providers.<name>.params]
temperature = 0
max_tokens = 4096

[models.providers.<name>.params."gpt-4o"]
temperature = 0.7
```

**配置键说明：**

<ResponseField name="models" type="string[]">
    在该提供商的交互式 `/model` 切换器中显示的模型名称列表。对于已自带模型配置文件的提供商，你在此处添加的名称会与内置名称并列显示——适用于尚未被添加到包中的新发布模型。对于[自定义提供商](#arbitrary-providers)，此列表是切换器中模型的唯一来源。

    此处列出的模型会**绕过**基于配置文件的[过滤条件](/oss/deepagents/cli/providers#which-models-appear-in-the-switcher)，始终显示在切换器中。这使其成为展示那些因配置文件缺少 `tool_calling` 支持或尚不存在而被排除的模型的推荐方式。

    此键为可选项。无论模型是否出现在切换器中，你始终可以通过 `/model` 或 `--model` 直接传入任何模型名称；提供商会在请求时验证名称。
</ResponseField>

<ResponseField name="api_key_env" type="string">
    可选，覆盖用于检查凭证的环境变量名称。
</ResponseField>

<ResponseField name="base_url" type="string">
    可选，覆盖提供商使用的基础 URL（如果支持）。请参阅你的提供商包的[参考文档](https://reference.langchain.com/python/integrations/)获取更多信息。
</ResponseField>

<ResponseField name="params" type="object">
    转发给模型构造函数的额外关键字参数。扁平键（如 `temperature = 0`）适用于该提供商的每个模型。模型键子表（如 `[params."gpt-4o"]`）仅覆盖该模型的个别值；合并为浅合并（冲突时模型级优先）。
</ResponseField>

<ResponseField name="profile" type="object">
    （高级）覆盖模型运行时[配置文件](/oss/langchain/models#model-profiles)中的字段（如 `max_input_tokens`）。扁平键适用于该提供商的每个模型。模型键子表（如 `[profile."claude-sonnet-4-5"]`）仅覆盖该模型的个别值；合并为浅合并（冲突时模型级优先）。这些覆盖在模型创建后应用，因此对上下文限制显示、自动摘要以及任何读取配置文件的功能都会生效。
</ResponseField>

<ResponseField name="class_path" type="string">
    用于[自定义模型](#arbitrary-providers)提供商。可选的完全限定 Python 类，格式为 `module.path:ClassName`。设置后，CLI 将直接为提供商 `<name>` 导入并实例化此类。该类必须是 `BaseChatModel` 的子类。
</ResponseField>

### 模型构造函数参数

任何提供商都可以使用 `params` 表向模型构造函数传递额外参数：

```toml
[models.providers.ollama.params]
temperature = 0
num_ctx = 8192
```

#### 按模型覆盖

如果特定模型需要不同的参数，可以在 `params` 下添加按模型键命名的子表来覆盖个别值，无需重复整个提供商配置：

```toml
[models.providers.ollama]
models = ["qwen3:4b", "llama3"]

[models.providers.ollama.params]
temperature = 0
num_ctx = 8192

[models.providers.ollama.params."qwen3:4b"]
temperature = 0.5
num_ctx = 4000
```

使用此配置：

* `ollama:qwen3:4b` 获得 `{temperature: 0.5, num_ctx: 4000}` —— 模型级覆盖优先。
* `ollama:llama3` 获得 `{temperature: 0, num_ctx: 8192}` —— 无覆盖，仅使用提供商级参数。

合并为浅合并：模型子表中存在的任何键会替换提供商级参数中的同名键，而仅存在于提供商级别的键则被保留。

#### 通过 `--model-params` 进行 CLI 覆盖

对于无需编辑配置文件的一次性调整，可以在启动时或使用 `/model` 命令在会话中通过 `--model-params` 传递 JSON 对象：

```bash
deepagents --model ollama:llama3 --model-params '{"temperature": 0.9, "num_ctx": 16384}'

# 非交互模式
deepagents -n "Summarize this repo" --model ollama:llama3 --model-params '{"temperature": 0}'
```

```txt title="在 TUI 中"
/model --model-params '{"temperature": 0.9}' ollama:llama3
/model --model-params '{"num_ctx": 16384}'  # 打开选择器，将参数应用于所选模型
```

这些参数具有最高优先级，会覆盖配置文件中的参数值。会话中设置的参数仅适用于当前会话，不会被持久化。`--model-params` 不能与 `--default` 组合使用。

### 配置文件覆盖

**（高级）**

覆盖模型运行时配置文件中的字段，以改变 CLI 对模型能力的解读方式。最常见的用例是降低 `max_input_tokens` 以更早触发自动摘要——这对测试或限制上下文使用量非常有用：

```toml
# 应用于该提供商的所有模型
[models.providers.anthropic.profile]
max_input_tokens = 4096
```

按模型子表的工作方式与 `params` 相同——冲突时模型级值优先：

```toml
[models.providers.anthropic.profile]
max_input_tokens = 4096

# 此模型获得更高的限制
[models.providers.anthropic.profile."claude-sonnet-4-5"]
max_input_tokens = 8192
```

配置文件覆盖在模型创建后合并到模型的配置文件中。任何读取配置文件的功能——状态栏中的上下文限制显示、自动摘要阈值、能力检查——都会看到被覆盖后的值。

#### 通过 `--profile-override` 进行 CLI 配置文件覆盖

**（高级）**

要在运行时覆盖模型配置文件字段而不编辑配置文件，可通过 `--profile-override` 传递 JSON 对象：

```bash
deepagents --profile-override '{"max_input_tokens": 4096}'

# 与 --model 组合使用
deepagents --model anthropic:claude-sonnet-4-5 --profile-override '{"max_input_tokens": 4096}'

# 非交互模式
deepagents -n "Summarize this repo" --profile-override '{"max_input_tokens": 4096}'
```

这些覆盖会合并到配置文件的 profile 覆盖之上（CLI 优先）。优先级链为：模型默认值 < config.toml profile < CLI `--profile-override`。

`--profile-override` 值在会话中切换模型时（`/model` 热切换）会持续生效——切换模型时会将覆盖重新应用于新模型。

### 自定义基础 URL

某些提供商包接受 `base_url` 来覆盖默认端点。例如，`langchain-ollama` 通过底层 `ollama` 客户端默认使用 `http://localhost:11434`。要将其指向其他地址，请在配置中设置 `base_url`：

```toml
[models.providers.ollama]
base_url = "http://your-host-here:port"
```

请参阅你的提供商参考文档了解兼容性信息和其他注意事项。

### 兼容 API

对于暴露与 OpenAI 或 Anthropic 线路兼容 API 的提供商，你可以通过将 `base_url` 指向提供商的端点来使用现有的 `langchain-openai` 或 `langchain-anthropic` 包：

```toml
[models.providers.openai]
base_url = "https://api.example.com/v1"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"]
```

```toml
[models.providers.anthropic]
base_url = "https://api.example.com"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"]
```

<Note>
    提供商在官方规范之上添加的任何功能都不会被捕获。如果提供商提供了专用的 LangChain 集成包，建议优先使用该包。
</Note>

### 向交互式切换器添加模型

某些提供商（如 `langchain-ollama`）不包含模型配置文件数据（完整列表请参阅[提供商参考](/oss/deepagents/cli/providers#provider-reference)）。在这种情况下，交互式 `/model` 切换器不会列出该提供商的模型。你可以通过在配置文件中为该提供商定义 `models` 列表来填补这一空白：

```toml
[models.providers.ollama]
models = ["llama3", "mistral", "codellama"]
```

`/model` 切换器现在将包含一个 Ollama 区域，列出这些模型。

这完全是可选的。你始终可以通过直接指定完整名称来切换到任何模型：

```txt
/model ollama:llama3
```

### 自定义提供商

你可以使用任何 [LangChain `BaseChatModel`](https://reference.langchain.com/python/langchain_core/language_models/#langchain_core.language_models.BaseChatModel) 子类，通过 `class_path` 来实现。CLI 将直接导入并实例化它：

```toml
[models.providers.my_custom]
class_path = "my_package.models:MyChatModel"
api_key_env = "MY_API_KEY"
base_url = "https://my-endpoint.example.com"

[models.providers.my_custom.params]
temperature = 0
max_tokens = 4096
```

包必须安装在与 `deepagents-cli` 相同的 Python 环境中：

```bash
# 如果 deepagents-cli 是通过 uv tool 安装的：
uv tool upgrade deepagents-cli --with my_package
```

当你切换到 `my_custom:my-model-v1`（通过 `/model` 或 `--model`）时，模型名称（`my-model-v1`）将作为 `model` 关键字参数传递：

```python
MyChatModel(model="my-model-v1", base_url="...", api_key="...", temperature=0, max_tokens=4096)
```

<Warning>
    `class_path` 会从你的配置文件中执行任意 Python 代码。这与 `pyproject.toml` 构建脚本的信任模型相同——你控制自己的机器。
</Warning>

你的提供商包可以选择在 `<package>.data._profiles` 中的 `_PROFILES` 字典中提供模型配置文件，以替代在 `models` 键下定义它们。更多信息请参阅 LangChain [模型配置文件](https://github.com/langchain-ai/langchain/tree/master/libs/model-profiles)。

## 钩子

钩子让外部程序能够响应 CLI 生命周期事件。在 `~/.deepagents/hooks.json` 中配置命令，CLI 将在事件触发时通过管道将 JSON 负载发送到每个匹配命令的 stdin。

钩子在后台线程中以"即发即忘"的方式运行——它们永远不会阻塞 CLI，失败也会被记录日志而不会中断你的会话。

### 设置

创建 `~/.deepagents/hooks.json`：

```json
{
  "hooks": [
    {
      "command": ["bash", "-c", "cat >> ~/deepagents-events.log"],
      "events": ["session.start", "session.end"]
    }
  ]
}
```

现在每次会话开始或结束时，CLI 都会将事件负载追加到 `~/deepagents-events.log`。

### 钩子配置

配置文件包含单个 `hooks` 数组。每个条目包含：

| 字段 | 类型 | 必填 | 说明 |
|-------|------|----------|-------------|
| `command` | `list[str]` | 是 | 要运行的命令及参数（不进行 shell 展开——如需要请使用 `["bash", "-c", "..."]`） |
| `events` | `list[str]` | 否 | 要订阅的事件名称。省略或留空则接收**所有**事件 |

```json
{
  "hooks": [
    {
      "command": ["python3", "my_handler.py"],
      "events": ["session.start", "task.complete"]
    },
    {
      "command": ["bash", "log_everything.sh"]
    }
  ]
}
```

上面的第二个钩子没有 `events` 过滤器，因此它会接收 CLI 发出的每个事件。

### 负载格式

每个钩子命令通过 stdin 接收一个带有 `"event"` 键以及事件特定字段的 JSON 对象：

```json
{
  "event": "session.start",
  "thread_id": "abc123"
}
```

### 事件参考

#### `session.start`

当 Agent 会话开始时触发（交互式和非交互式模式均适用）。

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `thread_id` | `string` | 会话线程标识符 |

#### `session.end`

当会话退出时触发。

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `thread_id` | `string` | 会话线程标识符 |

#### `user.prompt`

在交互模式下用户提交聊天消息时触发。

无额外字段。

#### `input.required`

当 Agent 需要人工输入（人机交互中断）时触发。

无额外字段。

#### `permission.request`

在一个或多个工具调用需要用户权限时，审批对话框出现前触发。

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `tool_names` | `list[str]` | 请求审批的工具名称 |

#### `tool.error`

当工具调用返回错误时触发。

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `tool_names` | `list[str]` | 出错的工具名称 |

#### `task.complete`

当 Agent 完成当前任务（流式循环结束且没有更多中断）时触发。

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `thread_id` | `string` | 会话线程标识符 |

#### `context.compact`

在 CLI 压缩（摘要）对话上下文之前触发。

无额外字段。

### 执行模型

- **后台线程**：钩子子进程通过 `asyncio.to_thread` 在线程中运行，因此主事件循环永远不会被阻塞。
- **并发调度**：当多个钩子匹配一个事件时，它们在线程池中并发运行。
- **5 秒超时**：每个命令有 5 秒超时限制。超时的命令会被终止。
- **即发即忘**：错误按钩子单独捕获，并以 debug/warning 级别记录日志。失败的钩子永远不会导致 CLI 崩溃或卡住。
- **延迟加载**：配置文件在首次事件调度时读取一次，并在会话剩余时间内缓存。
- **无 shell 展开**：命令直接执行（不通过 shell）。如需 shell 功能（如管道或变量展开），请用 `["bash", "-c", "..."]` 包装。

### 钩子示例

#### 将所有事件记录到文件

```json
{
  "hooks": [
    {
      "command": ["bash", "-c", "jq -c . >> ~/.deepagents/hook-events.jsonl"],
      "events": []
    }
  ]
}
```

#### 任务完成时发送桌面通知（macOS）

```json
{
  "hooks": [
    {
      "command": [
        "bash", "-c",
        "osascript -e 'display notification \"Agent finished\" with title \"Deep Agents\"'"
      ],
      "events": ["task.complete"]
    }
  ]
}
```

#### Python 处理器

编写一个从 stdin 读取 JSON 负载的处理器脚本：

```python title="my_handler.py"
import json
import sys

payload = json.load(sys.stdin)
event = payload["event"]

if event == "session.start":
    print(f"Session started: {payload['thread_id']}", file=sys.stderr)
elif event == "permission.request":
    print(f"Approval needed for: {payload['tool_names']}", file=sys.stderr)
```

```json title="~/.deepagents/hooks.json"
{
  "hooks": [
    {
      "command": ["python3", "my_handler.py"],
      "events": ["session.start", "permission.request"]
    }
  ]
}
```

### 安全注意事项

钩子遵循与 Git 钩子或 shell 别名相同的信任模型——任何可以写入 `~/.deepagents/hooks.json` 的用户都可以执行任意命令。这是设计使然：

- **无命令注入**：负载数据仅以 JSON 格式流向 stdin，永远不会流向命令行参数。`json.dumps` 处理转义。
- **默认无 shell**：命令以 `shell=False` 运行，防止 shell 注入。
- **格式错误的配置**：无效的 JSON 或意外类型会产生日志警告，而非安全问题。

<Warning>
    仅添加来自你信任的来源的钩子。钩子拥有与你的用户账户相同的权限。
</Warning>
