Deep Agents CLI 是一个基于 [Deep Agents SDK](/oss/deepagents/quickstart) 构建的开源终端编码 Agent。
它保持持久记忆、跨会话维护上下文、学习项目规范、使用可自定义的技能，并通过审批控制执行代码。

![Deep Agents CLI](/oss/images/deepagents/deepagents-cli.png)

Deep Agents CLI 具有以下内置功能：

* <Icon icon="file" size={16} /> **文件操作** - 读取、写入和编辑项目中的文件，通过工具让 Agent 能够管理和修改代码和文档。
* <Icon icon="terminal" size={16} /> **Shell 命令执行** - 执行 shell 命令以运行测试、构建项目、管理依赖以及与版本控制系统交互。
* <Icon icon="search" size={16} /> **网络搜索** - 搜索网络获取最新信息和文档（需要 Tavily API 密钥）。
* <Icon icon="globe" size={16} /> **HTTP 请求** - 向 API 和外部服务发起 HTTP 请求，用于数据获取和集成任务。
* <Icon icon="list-check" size={16} /> **任务规划和跟踪** - 将复杂任务分解为离散步骤，并通过内置的待办系统跟踪进度。
* <Icon icon="brain" size={16} /> **记忆存储与检索** - 跨会话存储和检索信息，使 Agent 能够记住项目规范和学习到的模式。
* <Icon icon="user" size={16} /> **人工干预** - 对敏感工具操作要求人工审批。
* <Icon icon="puzzle" size={16} /> **技能** - 通过存储在技能目录中的自定义专业知识和指令扩展 Agent 能力。

<Accordion title="内置工具完整列表">
    ## 内置工具

    Agent 自带以下内置工具，无需配置即可使用：

    | Tool | Description | Human-in-the-Loop |
    |------|-------------|-------------------|
    | `ls` | List files and directories | - |
    | `read_file` | Read contents of a file; supports images (`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`) as multimodal content | - |
    | `write_file` | Create or overwrite a file | Required<sup>1</sup> |
    | `edit_file` | Make targeted edits to existing files | Required<sup>1</sup> |

    | `grep` | Search for text patterns across files | - |
    | `shell` | Execute shell commands (local mode) | Required<sup>1</sup> |

    ```bash
    deepagents --auto-approve
    ```
</Accordion>

## 快速开始

<Steps>
    <Step title="设置模型凭据" icon="key">
        选择你要使用的模型提供商，并将凭据设置为环境变量。

        <Tabs>
            <Tab title="OpenAI">
                导出为环境变量：

                ```bash
                export OPENAI_API_KEY="your-api-key"
                ```
            </Tab>
            <Tab title="Anthropic">
                导出为环境变量：

                ```bash
                export ANTHROPIC_API_KEY="your-api-key"
                ```
            </Tab>
            <Tab title="Google">
                导出为环境变量：

                ```bash
                export GOOGLE_API_KEY="your-api-key"
                ```
            </Tab>
            <Tab title="Vertex AI">
                导出你的 Google Cloud 项目：

                ```bash
                export GOOGLE_CLOUD_PROJECT="your-gcp-project-id"
                ```

                Vertex AI 使用应用默认凭据（ADC）进行身份验证。
            </Tab>
        </Tabs>
    </Step>

    <Step title="运行 CLI" icon="terminal">
        <CodeGroup>
            ```bash Install globally
            uv tool install deepagents-cli
            deepagents
            ```

            ```bash Run (without global install)
            uvx deepagents-cli
            ```
        </CodeGroup>
    </Step>

    <Step title="给 Agent 一个任务" icon="message">
        ```txt
        > Create a Python script that prints "Hello, World!"
        ```

        Agent 会在修改文件前以 diff 形式提议更改，供你审批。
    </Step>
</Steps>

<Accordion title="其他安装和配置选项">
每个模型提供商需要安装其对应的 LangChain 集成包。安装 CLI 时可以作为可选额外包使用：

```bash
# Install with one provider
uv tool install 'deepagents-cli[anthropic]'

# Install with multiple providers at once
uv tool install 'deepagents-cli[ollama,groq]'

# Add additional packages at a later date
uv tool upgrade deepagents-cli --with langchain-xai
```

关于支持的提供商和配置选项的完整列表，请参见[自定义模型提供商](/oss/deepagents/cli/providers)。

首次启动时（未设置 `[models].default` 或 `[models].recent`），CLI 会按以下顺序自动选择第一个可用的启动凭据：`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`，然后是 `GOOGLE_CLOUD_PROJECT`（Vertex AI）。此启动回退机制有意限定范围；其他支持的提供商（如 Groq）仍可通过 `--model`、`/model` 或保存的默认模型使用。

```bash
deepagents --model anthropic:claude-opus-4-5
```

启用网络搜索（可选）：

```bash
export TAVILY_API_KEY="your-key"
```

API 密钥可以设置为环境变量或写在 `.env` 文件中。
</Accordion>

## 使用 LangSmith 追踪

启用 LangSmith 追踪以在 LangSmith 仪表板中查看 Agent 操作：

1. 启用 LangSmith 追踪：

   ```bash
   export LANGCHAIN_TRACING=true
   export LANGCHAIN_API_KEY="your-api-key"
   ```

1. 为 Deep Agent 操作（如工具调用和 Agent 决策）配置 Agent 追踪：

   ```bash
   export DEEPAGENTS_LANGSMITH_PROJECT="my-deep-agent-execution"
   ```

1. 如果你正在使用 Deep Agent 构建 LangChain 应用，并希望将 Agent 追踪与应用追踪分开，还可以配置 `LANGSMITH_PROJECT`：

   ```bash
   export LANGSMITH_PROJECT="my-app-calls-to-langchain"
   ```

配置后，CLI 显示：

```sh
✓ LangSmith tracing: 'my-project'
```

## 配置

每个 Agent 有自己的配置目录，位于 `~/.deepagents/<agent_name>/`。
默认 Agent 名称是 `agent`。
启动 Agent 时，如果该 Agent 名称的文件夹不存在，则会在 `~/.deepagents/<new_agent>/` 路径创建。

```bash
# List all configured agents
deepagents list

# Use a specific agent configuration
deepagents --agent mybot

# Use a specific model (provider:model format or auto-detect)
deepagents --model anthropic:claude-sonnet-4-5
deepagents --model gpt-4o

# Auto-approve tool usage (skip human-in-the-loop prompts)
deepagents --auto-approve

# Execute code in a remote sandbox
deepagents --sandbox modal        # or runloop, daytona
deepagents --sandbox-id dbx_123   # reuse existing sandbox
```

        | `--default-model [MODEL]` | Set the default model |
        | `--clear-default-model` | Clear the default model |
        | `-r`, `--resume [ID]`  | Resume a session: `-r` for most recent, `-r <ID>` for a specific thread |

        | `-n`, `--non-interactive TEXT` | Run a single task non-interactively and exit. Shell is disabled unless `--shell-allow-list` is set |
        | `-q`, `--quiet`        | Clean output for piping — only the agent's response goes to stdout. Requires `-n` or piped stdin |
        | `--no-stream`          | Buffer the full response and write to stdout at once instead of streaming. Requires `-n` or piped stdin |

        | `--sandbox-setup PATH` | Path to setup script to run in sandbox after creation       |

## 交互模式

像聊天界面一样自然地输入。
Agent 将使用其内置工具、技能和记忆来帮助你完成任务。

<AccordionGroup>
    <Accordion title="斜杠命令" icon="slash">
        在 CLI 会话中使用以下命令：

        - `/model` - Open the interactive model selector
        - `/model <provider:model>` - Switch directly to a specific model (e.g., `/model anthropic:claude-sonnet-4-5`)
        - `/model --default <provider:model>` - Set a persistent default model
        - `/model --default --clear` - Clear the saved default model
        - `/remember [context]` - Review conversation and update memory and skills. Optionally pass additional context
        - `/tokens` - Display current context window token usage
        - `/clear` - Clear conversation history and start a new thread
        - `/threads` - Browse and resume previous conversation threads
        - `/trace` - Open the current thread in LangSmith (requires `LANGSMITH_API_KEY`)
        - `/changelog` - Open the CLI changelog in your browser
        - `/docs` - Open the documentation in your browser
        - `/feedback` - Open the GitHub issues page to file a bug report or feature request
        - `/version` - Show installed `deepagents-cli` and SDK versions
        - `/help` - Show help and available commands
        - `/quit` (or `/q`) - Exit the CLI
    </Accordion>

## 非交互模式和管道

使用 `-n` 运行单个任务，无需启动交互界面：

```bash
deepagents -n "Write a Python script that prints hello world"
```

你也可以通过 stdin 管道输入。当输入被管道传入时，CLI 自动以非交互模式运行：

```bash
echo "Explain this code" | deepagents
cat error.log | deepagents -n "What's causing this error?"
git diff | deepagents -n "Review these changes"
```

当管道输入与 `-n` 或 `-m` 组合使用时，管道内容会被前置到参数值之前。

<Note>
    管道输入的最大大小为 10 MiB。
</Note>

使用 `-q` 获取适合管道传递给其他命令的干净输出，使用 `--no-stream` 缓冲完整响应（而非流式输出）后再写入 stdout：

```bash
deepagents -n "Generate a .gitignore for Python" -q > .gitignore
deepagents -n "List dependencies" -q --no-stream | sort
```

默认情况下，非交互模式下 shell 执行被禁用。使用 `--shell-allow-list` 允许特定命令：

```bash
deepagents -n "Run the tests and fix failures" --shell-allow-list "pytest,git,make"
deepagents -n "Build the project" --shell-allow-list recommended
```

## 切换模型

你可以在会话期间使用 `/model` 命令切换模型，无需重启 CLI，或在启动时通过 `--model` 标志指定：

```txt
> /model anthropic:claude-opus-4-5
> /model openai:gpt-4o
```

```bash
deepagents --model openai:gpt-4o
```

不带参数运行 `/model` 可打开交互式模型选择器，按提供商分组显示可用模型。

关于切换模型、设置默认值、配置自定义提供商以及使用 `~/.deepagents/config.toml` 配置文件的完整详情，请参见[自定义模型提供商](/oss/deepagents/cli/providers)。

## 教 Agent 学习项目规范

随着你使用 Agent，它会自动将信息以 markdown 文件的形式存储在 `~/.deepagents/<agent_name>/memories/` 中，采用记忆优先协议：

1. **研究**：在开始任务前搜索记忆中的相关上下文
2. **响应**：在执行过程中不确定时检查记忆
3. **学习**：自动保存新信息供未来会话使用

Agent 按主题组织记忆，使用描述性文件名：

```
~/.deepagents/backend-dev/memories/
├── api-conventions.md
├── database-schema.md
└── deployment-process.md
```

当你教 Agent 规范时：

```bash
uvx deepagents-cli --agent backend-dev
> Our API uses snake_case and includes created_at/updated_at timestamps
```

它会在未来的会话中记住：

```bash
> Create a /users endpoint
# Applies conventions without prompting
```

## 自定义你的 Deep Agent

自定义任何 Agent 有两种主要方式：

- **记忆**：全局和项目级别的 `AGENTS.md` 文件，在会话开始时完整加载。
  用于通用编码风格和偏好。

- **技能**：全局和项目级别的上下文、规范、指南或指令。
  用于仅在执行特定任务时才需要的上下文。

### 提供项目或用户上下文

[`AGENTS.md` 文件](https://agents.md/) 提供持久记忆，始终在会话开始时加载。

你可以在 `~/.deepagents/<agent_name>/AGENTS.md` 中为 Agent 提供全局用户记忆。
此文件在你启动新的 Deep Agent 会话时始终被加载。
Agent 也可能在回答项目特定问题或你引用过往工作或模式时读取其记忆文件。

对于项目级别的记忆，你可以在任何项目的根目录下的 `.deepagents/AGENTS.md` 中添加上下文，前提是该项目使用 git。
当你从项目文件夹内的任何位置启动 CLI 时，CLI 会通过检查包含 `.git` 文件夹来查找项目根目录。

全局和项目级别的 `AGENTS.md` 文件会一起加载，并在启动时追加到系统提示词中。
Agent 会随着你的使用更新它们，并根据你提供的关于行为方式的额外信息、对其工作的反馈或记忆指令来更新。
Agent 还会在从你的交互中识别到模式或偏好时更新记忆。

如果你希望明确提示 Deep Agent 根据当前线程的上下文更新技能和记忆，使用 `/remember` 命令，它会加载一个自定义指令来审查上下文并执行更新。

要在额外记忆文件中添加更结构化的项目知识，你可以将它们添加在 `.deepagents/` 中并在 `AGENTS.md` 文件中引用它们。
你必须在 `AGENTS.md` 文件中引用额外文件，Agent 才能知道这些文件的存在。
额外文件不会在启动时读取，但 Agent 可以在需要时引用和更新它们。

<Accordion title="何时使用全局 vs. 项目 AGENTS.md">
**全局 `AGENTS.md`**（`~/.deepagents/agent/AGENTS.md`）

- 你的个性、风格和通用编码偏好
- 通用的语气和沟通风格
- 通用编码偏好（格式化、类型提示等）
- 适用于所有场景的工具使用模式
- 不随项目变化的工作流程和方法论

**项目 `AGENTS.md`**（项目根目录中的 `.deepagents/AGENTS.md`）

- 项目特定的上下文和规范
- 项目架构和设计模式
- 此代码库特定的编码规范
- 测试策略和部署流程
- 团队指南和项目结构
</Accordion>

## 使用远程沙箱

在隔离的远程环境中执行代码，以确保安全性和灵活性。远程沙箱提供以下优势：

  - **安全性**：保护本地机器免受潜在有害代码执行的影响
  - **干净环境**：无需本地设置即可使用特定的依赖或操作系统配置
  - **并行执行**：在隔离环境中同时运行多个 Agent
  - **长时间任务**：执行耗时操作而不会阻塞你的机器
  - **可复现性**：确保团队间执行环境的一致性

要使用远程沙箱，请执行以下步骤：

1. 配置你的沙箱提供商（[Runloop](https://www.runloop.ai/)、[Daytona](https://www.daytona.io/) 或 [Modal](https://modal.com/)）：

    ```bash
    # Runloop
    export RUNLOOP_API_KEY="your-key"

    # Daytona
    export DAYTONA_API_KEY="your-key"

    # Modal
    modal setup
    ```

1. 使用沙箱运行 CLI：

    ```bash
    uvx deepagents-cli --sandbox runloop --sandbox-setup ./setup.sh
    ```

    Agent 在本地运行，但在远程沙箱中执行所有代码操作。可选的设置脚本可以配置环境变量、克隆仓库和准备依赖。

1. （可选）创建 `setup.sh` 文件来配置沙箱环境：

    ```bash
    #!/bin/bash
    set -e

    # Clone repository using GitHub token
    git clone https://x-access-token:${GITHUB_TOKEN}@github.com/username/repo.git $HOME/workspace
    cd $HOME/workspace

    # Make environment variables persistent
    cat >> ~/.bashrc <<'EOF'
    export GITHUB_TOKEN="${GITHUB_TOKEN}"
    export OPENAI_API_KEY="${OPENAI_API_KEY}"
    cd $HOME/workspace
    EOF

    source ~/.bashrc
    ```

    将密钥存储在本地 `.env` 文件中供设置脚本访问。

<Warning>
    沙箱隔离了代码执行，但 Agent 对不受信任的输入仍然易受提示注入攻击。仅使用人工干预审批、短期密钥和受信任的设置脚本。

    注意沙箱 API 正在快速演进，我们预计更多提供商将支持有助于缓解提示注入和密钥管理问题的代理。
</Warning>

## 使用技能

技能是可复用的 Agent 能力，提供专业工作流和领域知识。
你可以使用[技能](/oss/deepagents/skills)为 Deep Agent 提供新的能力和专业知识。
Deep Agent 技能遵循 [Agent Skills 标准](https://agentskills.io/)。
添加技能后，Deep Agent 会自动使用它们，并随着你的使用和提供的额外信息进行更新。

如果你希望明确提示 Deep Agent 根据当前线程的上下文更新技能和记忆，使用 `/remember` 命令，它会加载一个自定义指令来审查上下文并执行更新。

### 添加技能

1. 首先创建一个技能：

   <Tabs>
       <Tab title="用户技能">
           ```bash
           deepagents skills create test-skill
           ```

           这将在你的 `~/.deepagents/<agent_name>` 文件夹中生成以下文件：
       </Tab>
       <Tab title="项目技能">
           ```bash
           deepagents skills create test-skill --project
           ```

           这将在你的 `~/{project}/.deepagents/skills` 文件夹中生成以下文件：
       </Tab>
   </Tabs>

   ```plaintext
   skills/
   └── test-skill
       └── SKILL.md
   ```

1. 打开生成的 `SKILL.md` 并编辑文件以包含你的指令。

1. 可选地将额外的脚本或其他资源添加到 `test-skill` 文件夹中。
   更多信息请参见[示例](/oss/deepagents/skills#examples)。

如果你已经有包含技能文件的技能文件夹，也可以直接复制到 Agent 的文件夹：

```bash
mkdir -p ~/.deepagents/<agent_name>/skills
cp -r examples/skills/web-research ~/.deepagents/<agent_name>/skills/
```

启动时，CLI 会从 Deep Agents 和共享别名目录中发现技能：

```text
`~/.deepagents/<agent_name>/skills/`
`~/.agents/skills/`
`.deepagents/skills/`
`.agents/skills/`.
```
当存在重名技能时，后置优先级目录会覆盖前置目录（参见[应用数据](/oss/deepagents/data-locations#skills)）。

对于项目级别的技能，项目的根目录必须有 `.git` 文件夹。
当你从项目文件夹内的任何位置启动 CLI 时，CLI 会通过检查包含 `.git` 文件夹来查找项目根目录。

对于每个技能，CLI 从 `SKILL.md` 文件的 frontmatter 中读取名称和描述。
当你使用 CLI 时，如果任务与技能的描述匹配，Agent 会读取该技能文件并遵循其指令。

### 列出技能

要查看你已安装的技能列表，运行：

<Tabs>
    <Tab title="用户技能">
        ```bash
        deepagents skills list
        ```
    </Tab>
    <Tab title="项目技能">
        ```bash
        deepagents skills list --project
        ```
    </Tab>
</Tabs>

要获取特定技能的更多信息，运行：

<Tabs>
    <Tab title="用户技能">
        ```bash
        deepagents skills info test-skill
        ```
    </Tab>
    <Tab title="项目技能">
        ```bash
        deepagents skills info test-skill --project
        ```
    </Tab>
</Tabs>
