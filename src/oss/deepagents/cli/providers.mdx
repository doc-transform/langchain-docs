Deep Agents CLI 支持任何[与 LangChain 兼容的聊天模型提供商](/oss/langchain/models)，几乎可以解锁任何支持工具调用的 LLM。任何暴露 OpenAI 兼容或 Anthropic 兼容 API 的服务也可开箱即用——请参见[兼容 API](#compatible-apis)。

## 快速开始

CLI 可与[以下模型提供商](#provider-reference)自动集成——除了安装相关提供商包外无需额外配置。

1. **安装提供商包**

    每个模型提供商需要安装其对应的 LangChain 集成包。安装 CLI 时可以作为可选额外包使用：

    ```bash
    # Install with one provider
    uv tool install 'deepagents-cli[anthropic]'

    # Install with multiple providers at once
    uv tool install 'deepagents-cli[anthropic,openai,groq]'

    # Add additional packages at a later date
    uv tool upgrade deepagents-cli --with langchain-ollama

    # All providers
    uv tool install 'deepagents-cli[anthropic,bedrock,cohere,deepseek,fireworks,google-genai,groq,huggingface,ibm,mistralai,nvidia,ollama,openai,openrouter,perplexity,vertexai,xai]'
    ```

2. **设置 API 密钥**

    大多数提供商需要 API 密钥。设置下表中列出的相应环境变量。每个集成包的详细信息请参见其文档。

### 切换模型

要在 CLI 中切换模型，可以：

1. **使用交互式模型切换器**，通过 `/model` 命令。它会显示一个硬编码的已知模型配置文件列表，来源于每个 LangChain 提供商包。

    <Note>
        注意这些配置文件并非可用模型的完整列表。如果想要的模型未显示，请使用选项 2（适用于新发布但尚未添加到配置文件中的模型）。
    </Note>
2. **直接指定模型名称**作为参数，例如 `/model openai:gpt-4o`。无论模型是否在选项 1 的列表中显示，你都可以使用所选提供商支持的任何模型。模型名称会被传递到 API 请求中。
3. **在启动时指定模型**，通过 `--model`，例如：

    ```txt
    deepagents --model openai:gpt-4o
    ```

### 设置默认模型

你可以设置一个持久默认模型，用于所有未来的 CLI 启动：

* **通过模型选择器：**打开 `/model`，导航到所需模型，然后按 `Ctrl+S` 将其固定为默认。再次在当前默认模型上按 `Ctrl+S` 可清除默认值。
* **通过命令：**`/model --default provider:model`（例如，`/model --default anthropic:claude-opus-4-6`）
* **通过配置文件：**在 `~/.deepagents/config.toml` 中设置 `[models].default`（请参见[配置文件](#config-file)）。
* **通过 shell：**

    ```bash
    deepagents --default-model anthropic:claude-opus-4-6
    ```

查看当前默认值：

```bash
deepagents --default-model
```

清除默认值：

* **通过 shell：**

    ```bash
    deepagents --clear-default-model
    ```

* **通过命令：**`/model --default --clear`
* **通过模型选择器：**在当前固定的默认模型上按 `Ctrl+S`。

未设置默认值时，CLI 将默认使用最近使用的模型。

### 模型解析顺序

CLI 启动时，按以下顺序解析要使用的模型：

1. **`--model` 标志**——提供时始终优先。
2. **`[models].default`**（`~/.deepagents/config.toml` 中）——用户有意设置的长期偏好。
3. **`[models].recent`**（`~/.deepagents/config.toml` 中）——通过 `/model` 切换的最后一个模型。自动写入；永远不会覆盖 `[models].default`。
4. **环境自动检测**——回退到第一个拥有有效 API 密钥的提供商，按以下顺序检查：`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`、`GOOGLE_CLOUD_PROJECT`（Vertex AI）。

此启动回退机制有意仅检查这四个凭据。其他支持的提供商（例如 Groq）仍可通过 `--model`、`/model` 以及保存的默认值（`[models].default` / `[models].recent`）使用。

## 配置文件

Deep Agents CLI 支持通过 `~/.deepagents/config.toml` 扩展和修改单个模型和提供商配置。

每个提供商是 `[models.providers]` 命名空间下的一个 TOML 表：

```toml
[models.providers.<name>]
models = ["gpt-4o"]
api_key_env = "OPENAI_API_KEY"
base_url = "https://api.openai.com/v1"
class_path = "my_package.models:MyChatModel"

[models.providers.<name>.params]
temperature = 0
max_tokens = 4096

[models.providers.<name>.params."gpt-4o"]
temperature = 0.7
```

**参数说明：**

<ResponseField name="models" type="string[]">
    一个模型名称列表，显示在此提供商的交互式 `/model` 切换器中。对于已经自带模型配置文件的提供商，你在此添加的名称会与内置的一起显示——适用于新发布但尚未添加到包中的模型。对于[任意提供商](#arbitrary-providers)，此列表是切换器中模型的唯一来源。

    此键是可选的。无论模型是否在切换器中显示，你始终可以直接将任何模型名称传给 `/model` 或 `--model`；提供商会在请求时验证名称。
</ResponseField>

<ResponseField name="api_key_env" type="string">
    可选地覆盖用于检查凭据的环境变量名称。
</ResponseField>

<ResponseField name="base_url" type="string">
    可选地覆盖提供商使用的基础 URL（如果支持）。请参见你的提供商包的[参考文档](https://reference.langchain.com/python/integrations/)获取更多信息。
</ResponseField>

<ResponseField name="params" type="object">
    传递给模型构造函数的额外关键字参数。平坦键（如 `temperature = 0`）适用于此提供商的每个模型。模型键的子表（如 `[params."gpt-4o"]`）仅覆盖该模型的单个值；合并是浅合并的（模型级别在冲突时优先）。
</ResponseField>

<ResponseField name="profile" type="object">
    （高级）覆盖模型运行时[配置文件](/oss/langchain/models#model-profiles)中的字段（如 `max_input_tokens`）。平坦键适用于此提供商的每个模型。模型键的子表（如 `[profile."claude-sonnet-4-5"]`）仅覆盖该模型的单个值；合并是浅合并的（模型级别在冲突时优先）。这些覆盖在模型创建后应用，因此对上下文限制显示、自动摘要以及其他读取配置文件的功能都会生效。
</ResponseField>

<ResponseField name="class_path" type="string">
    用于[任意模型](#arbitrary-providers)提供商。可选的完全限定 Python 类，格式为 `module.path:ClassName`。设置后，CLI 会直接导入并实例化此类作为提供商 `<name>`。该类必须是 `BaseChatModel` 的子类。
</ResponseField>

你可以在 `~/.deepagents/config.toml` 中设置默认模型——可以直接编辑文件、使用 `/model --default`，或在交互式模型选择器中选择默认值。

```toml
[models]
default = "ollama:qwen3:4b"        # your intentional long-term preference
recent = "anthropic:claude-sonnet-4-5"  # last /model switch (written automatically)
```

`[models].default` 始终优先于 `[models].recent`。`/model` 命令仅写入 `[models].recent`，因此你配置的默认值不会被会话中的切换覆盖。要移除默认值，使用 `/model --default --clear` 或从配置文件中删除 `default` 键。

## 示例

### 模型构造参数

任何提供商都可以使用 `params` 表向模型构造函数传递额外参数：

```toml
[models.providers.ollama.params]
temperature = 0
num_ctx = 8192
```

#### 按模型覆盖

如果特定模型需要不同的参数，在 `params` 下添加模型键的子表来覆盖单个值，而无需复制整个提供商配置：

```toml
[models.providers.ollama]
models = ["qwen3:4b", "llama3"]

[models.providers.ollama.params]
temperature = 0
num_ctx = 8192

[models.providers.ollama.params."qwen3:4b"]
temperature = 0.5
num_ctx = 4000
```

使用此配置：

* `ollama:qwen3:4b` 获得 `{temperature: 0.5, num_ctx: 4000}`——模型覆盖优先。
* `ollama:llama3` 获得 `{temperature: 0, num_ctx: 8192}`——无覆盖，仅使用提供商级别参数。

合并是浅合并的：模型子表中存在的任何键会替换提供商级别参数中的同名键，而仅存在于提供商级别的键会被保留。

### Profile 覆盖

**（高级）**

覆盖模型运行时配置文件中的字段，以改变 CLI 对模型能力的解读方式。最常见的用例是降低 `max_input_tokens` 以更早触发自动摘要——适用于测试或限制上下文使用量：

```toml
# 应用于此提供商的所有模型
[models.providers.anthropic.profile]
max_input_tokens = 4096
```

按模型子表的工作方式与 `params` 相同——模型级别的值在冲突时优先：

```toml
[models.providers.anthropic.profile]
max_input_tokens = 4096

# 此模型获得更高的限制
[models.providers.anthropic.profile."claude-sonnet-4-5"]
max_input_tokens = 8192
```

Profile 覆盖在模型创建后合并到模型的配置文件中。任何读取配置文件的功能——状态栏中的上下文限制显示、自动摘要阈值、能力检查——都将看到覆盖后的值。

#### 通过 `--model-params` 进行 CLI 覆盖

如需一次性调整而不编辑配置文件，通过 `--model-params` 传递 JSON 对象：

```bash
deepagents --model ollama:llama3 --model-params '{"temperature": 0.9, "num_ctx": 16384}'

# In non-interactive mode
deepagents -n "Summarize this repo" --model ollama:llama3 --model-params '{"temperature": 0}'
```

这些参数具有最高优先级，会覆盖配置文件中的参数值。

#### 通过 `--profile-override` 进行 CLI Profile 覆盖

**（高级）**

如需在运行时覆盖模型 Profile 字段而不编辑配置文件，通过 `--profile-override` 传递 JSON 对象：

```bash
deepagents --profile-override '{"max_input_tokens": 4096}'

# 与 --model 组合使用
deepagents --model anthropic:claude-sonnet-4-5 --profile-override '{"max_input_tokens": 4096}'

# 非交互模式下
deepagents -n "Summarize this repo" --profile-override '{"max_input_tokens": 4096}'
```

这些值会合并到配置文件的 Profile 覆盖之上（CLI 优先）。优先级链为：模型默认值 < config.toml profile < CLI `--profile-override`。

### 指定自定义 `base_url`

某些提供商包接受 `base_url` 来覆盖默认端点。例如，`langchain-ollama` 通过底层 `ollama` 客户端默认使用 `http://localhost:11434`。要将其指向其他地址，在配置中设置 `base_url`：

```toml
[models.providers.ollama]
base_url = "http://your-host-here:port"
```

具体兼容性信息和其他注意事项请参见你的提供商的参考文档。

### 兼容 API

对于暴露与 OpenAI 或 Anthropic 协议兼容的 API 的提供商，你可以使用现有的 `langchain-openai` 或 `langchain-anthropic` 包，将 `base_url` 指向提供商的端点：

```toml
[models.providers.openai]
base_url = "https://api.example.com/v1"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"]
```

```toml
[models.providers.anthropic]
base_url = "https://api.example.com"
api_key_env = "EXAMPLE_API_KEY"
models = ["my-model"]
```

<Note>
    提供商在官方规范之上添加的任何功能都不会被捕获。如果提供商提供了专用的 LangChain 集成包，请优先使用该包。
</Note>

### 向交互式切换器添加模型

某些提供商（如 `langchain-ollama`）不自带模型配置文件数据（完整列表请参见[提供商参考](#provider-reference)）。这种情况下，交互式 `/model` 切换器不会列出该提供商的模型。你可以在配置文件中为该提供商定义 `models` 列表来填补此缺口：

```toml
[models.providers.ollama]
models = ["llama3", "mistral", "codellama"]
```

`/model` 切换器现在将包含一个 Ollama 部分，列出这些模型。

这完全是可选的。你始终可以通过直接指定完整名称切换到任何模型：

```txt
/model ollama:llama3
```

### 任意提供商

你可以使用任何 [LangChain `BaseChatModel`](https://reference.langchain.com/python/langchain_core/language_models/#langchain_core.language_models.BaseChatModel) 子类，通过 `class_path` 指定。CLI 会直接导入并实例化它：

```toml
[models.providers.my_custom]
class_path = "my_package.models:MyChatModel"
api_key_env = "MY_API_KEY"
base_url = "https://my-endpoint.example.com"

[models.providers.my_custom.params]
temperature = 0
max_tokens = 4096
```

该包必须安装在与 `deepagents-cli` 相同的 Python 环境中：

```bash
# If deepagents-cli was installed with uv tool:
uv tool upgrade deepagents-cli --with my_package
```

当你切换到 `my_custom:my-model-v1`（通过 `/model` 或 `--model`）时，模型名称（`my-model-v1`）会作为 `model` 关键字参数传递：

```python
MyChatModel(model="my-model-v1", base_url="...", api_key="...", temperature=0, max_tokens=4096)
```

<Warning>
`class_path` 会从你的配置文件执行任意 Python 代码。其信任模型与 `pyproject.toml` 构建脚本相同——你控制的是你自己的机器。
</Warning>

你的提供商包可以选择在 `<package>.data._profiles` 中的 `_PROFILES` 字典中提供模型配置文件，以代替在 `models` 键下定义它们。更多信息请参见 LangChain [模型配置文件](https://github.com/langchain-ai/langchain/tree/master/libs/model-profiles)。
