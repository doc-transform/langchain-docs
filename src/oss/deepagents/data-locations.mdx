Deep Agents CLI 将数据存储在两个目录层次中：

- **`~/.deepagents/`** — Deep Agents 专用数据（Agent 记忆、技能、会话）
- **`~/.agents/`** — 工具无关数据（跨 AI CLI 工具共享的技能）

## 目录结构

```text
~/.deepagents/
├── sessions.db              # 会话检查点的 SQLite 数据库
├── history.jsonl            # 命令输入历史
└── {agent}/                 # 每个 Agent 的目录（默认为 "agent"）
    ├── AGENTS.md            # 用户自定义的 Agent 指令
    ├── skills/              # 用户级别的技能
    │   └── {skill-name}/
    │       └── SKILL.md
    └── agents/              # 自定义子 Agent 定义
        └── {subagent-name}/
            └── AGENTS.md

~/.agents/                   # 工具无关别名（跨 AI CLI 共享）
└── skills/                  # 可用于任何兼容工具的技能
    └── {skill-name}/
        └── SKILL.md

{project}/                   # 项目级别（位于 git 仓库根目录）
├── AGENTS.md                # 项目指令（根级别）
└── .deepagents/
│   ├── AGENTS.md            # 项目指令（推荐位置）
│   ├── skills/              # 项目特定技能
│   │   └── {skill-name}/
│   │       └── SKILL.md
│   └── agents/              # 项目特定子 Agent
│       └── {subagent-name}/
│           └── AGENTS.md
└── .agents/                 # 工具无关的项目技能
    └── skills/
        └── {skill-name}/
            └── SKILL.md
```

## 各数据存储位置说明

| 数据 | 位置 | 读/写 | 说明 |
|------|----------|------------|-------|
| **会话** | `~/.deepagents/sessions.db` | 读/写 | SQLite 检查点数据库 |
| **输入历史** | `~/.deepagents/history.jsonl` | 读/写 | JSON-lines 格式，支持上下箭头快速调出 |
| **基础指令** | 包内 `default_agent_prompt.md` | 读 | 不可变，随 CLI 升级更新 |
| **用户自定义** | `~/.deepagents/{agent}/AGENTS.md` | 读/写 | 追加到基础指令后 |
| **项目指令** | `.deepagents/AGENTS.md` 或 `AGENTS.md` | 读 | 如果两者都存在，则两者都会加载 |
| **用户技能** | `~/.deepagents/{agent}/skills/` | 读/写 | Agent 专用技能 |
| **共享技能** | `~/.agents/skills/` | 读 | 工具无关，跨 CLI 共享 |
| **项目技能** | `.deepagents/skills/` 或 `.agents/skills/` | 读 | 项目作用域 |
| **自定义子 Agent** | `~/.deepagents/{agent}/agents/` | 读/写 | 用户定义的子 Agent |
| **项目子 Agent** | `.deepagents/agents/` | 读 | 项目定义的子 Agent |

## 优先级规则

当同一项存在于多个位置时，**优先级较高的完全胜出**（不合并）。

### 技能

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/skills/` — 用户级别 Deep Agents 技能
2. `~/.agents/skills/` — 用户级别工具无关技能
3. `.deepagents/skills/` — 项目级别 Deep Agents 技能
4. `.agents/skills/` — 项目级别工具无关技能 *（最高优先级）*

### 子 Agent

优先级顺序（从低到高）：

1. `~/.deepagents/{agent}/agents/` — 用户级别
2. `.deepagents/agents/` — 项目级别 *（最高优先级）*

### 指令

所有指令源会被**合并**（而非覆盖）：

1. 包内置基础提示词 *（始终加载）*
2. `~/.deepagents/{agent}/AGENTS.md` *（追加）*
3. `.deepagents/AGENTS.md` *（追加）*
4. 项目根目录的 `AGENTS.md` *（追加）*

## `.deepagents` 与 `.agents` 的区别

| 目录 | 用途 | 何时使用 |
|-----------|---------|-------------|
| `.deepagents/` | Deep Agents CLI 专用 | 使用 CLI 特有功能的技能和配置 |
| `.agents/` | 工具无关 | 希望跨不同 AI CLI 工具共享的技能 |

<Tip>
将可以与任何 AI 编码助手配合使用的技能放在 `.agents/skills/` 中。
将依赖 Deep Agents 特有工具或约定的技能放在 `.deepagents/skills/` 中。
</Tip>

## 清理数据

| 需求 | 操作 |
|------|--------|
| 重置所有数据 | `rm -rf ~/.deepagents` |
| 仅清除会话 | `rm ~/.deepagents/sessions.db` |
| 清除输入历史 | `rm ~/.deepagents/history.jsonl` |
| 重置 Agent 指令 | `deepagents reset --agent {name}` |
| 删除某个技能 | `rm -rf ~/.deepagents/{agent}/skills/{skill-name}` |

<Warning>
删除 `~/.deepagents/sessions.db` 将移除所有对话历史和检查点。
此操作不可撤销。
</Warning>
