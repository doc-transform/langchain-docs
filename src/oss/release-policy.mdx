    ## 发布节奏

    :::python
    随着 LangChain 1.0 的发布，`langchain` 和 `langchain-core` 的**次版本**发布（例如，从 `1.0.x` 到 `1.1.0`）遵循语义化版本控制，可能会频繁发布。次版本包含新功能和改进，但不包含破坏性变更。

    修订版发布较为频繁，每周可能发布多次，包含错误修复和小幅改进。

    ## API 稳定性

    :::python
    LLM 应用开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `langchain-core` 中的 API 将持续演进，以更好地满足用户需求。
    :::
    :::js
    LLM 应用开发是一个快速发展的领域，我们不断从用户和社区中学习。因此，我们预计 `langchain` 和 `@langchain/core` 中的 API 将持续演进，以更好地满足用户需求。
    :::

    随着 LangChain 1.0 采用语义化版本控制：

    - 公共 API 的破坏性变更仅会在主版本发布时发生（例如，`2.0.0`）
    - 次版本升级（例如，`1.0.0` 到 `1.1.0`）添加新功能且不包含破坏性变更
    - 修订版升级（例如，`1.0.0` 到 `1.0.1`）包含错误修复和小幅改进

    我们通常会避免进行不必要的变更，并为即将移除的功能提供废弃策略。

    ### 其他包的稳定性

    LangChain 生态系统中其他包的稳定性可能有所不同：

    - **LangChain 团队维护的合作伙伴包**（如 `langchain-openai` 和 `langchain-anthropic`）遵循语义化版本控制，在 1.0 发布后预期是稳定的。其他合作伙伴包可能遵循不同的稳定性和版本控制策略，用户应参考对应包的文档了解更多信息。

    :::python
    - **`langchain-community`** 是一个由社区维护的包，包含第三方集成。由于集成数量较多，`langchain-community` 不遵循与 `langchain` 和 `langchain-core` 相同的严格语义化版本控制策略。详见长期支持下的“特别说明”部分。
    :::
    :::js
    - **`@langchain/community`** 是一个由社区维护的包，包含第三方集成。由于社区贡献和第三方依赖的特性，`@langchain/community` 不遵循与 `langchain` 和 `@langchain/core` 相同的严格语义化版本控制策略。详见长期支持下的“特别说明”部分。
    :::

    ## 废弃策略

    我们通常会在更好的替代方案可用之前避免废弃功能。

    :::python
    随着 LangChain 1.0 采用语义化版本控制，已废弃的功能将在整个 1.x 版本系列中继续正常工作。破坏性变更（包括移除已废弃功能）仅会在主版本发布时发生（例如，2.0）。

    当 `langchain` 或 `langchain-core` 中的功能被废弃时，我们会：
    - 在代码和文档中明确标记为废弃
    - 提供向推荐替代方案的迁移指南
    - 在所有 1.x 次版本中为已废弃功能提供安全更新
    :::
    :::js
    随着 LangChain 1.0 采用语义化版本控制，已废弃的功能将在整个 1.x 版本系列中继续正常工作。破坏性变更（包括移除已废弃功能）仅会在主版本发布时发生（例如，2.0）。

    当 `langchain` 或 `@langchain/core` 中的功能被废弃时，我们会：
    - 在代码和文档中明确标记为废弃
    - 提供向推荐替代方案的迁移指南
    - 在所有 1.x 次版本中为已废弃功能提供安全更新

    在某些情况下，如果已废弃的功能不会引起维护问题，我们可能会允许它们在代码库中保留更长时间，以进一步减轻用户的迁移负担。

    ## 长期支持（LTS）

    LangChain 遵循长期支持（LTS）策略，为生产应用提供稳定性保障：

    ### 发布状态定义

    包会被标记为以下状态之一：
    - **ACTIVE**：当前活跃开发中，包含错误修复、安全补丁和新功能
    - **MAINTENANCE**：接收所有安全补丁和关键错误修复，但不再添加新功能

    ### 当前 LTS 版本

    **LangChain 1.0** 被指定为 LTS 版本：
    - **状态**：ACTIVE，直到 2.0 发布
    - **支持期限**：2.0 发布后，1.0 将进入 MAINTENANCE 模式，至少维持 1 年
    - **语义化版本控制匹配**：用户可以在次版本之间安全升级（例如，从 1.0 到 1.1）而不会遇到破坏性变更

    ### 旧版本支持

    **LangChain 0.3**：

    - **状态**：MAINTENANCE 模式
    - **支持期限**：至 2026 年 12 月
    - **支持范围**：安全补丁和关键错误修复

    ### 特别说明

    :::python
    **langchain-community 0.4**：由于社区贡献和第三方集成的特性，`langchain-community` 可能在次版本发布时包含破坏性变更。它以 0.4 版本发布以反映其不同的稳定性策略。
    :::
    :::js
    **@langchain/community 1.0**：由于社区贡献和第三方集成的特性，`@langchain/community` 可能在次版本发布时包含破坏性变更。
    :::

    <Tab title="LangGraph">
    LangGraph 遵循结构化的发布策略，为构建生产应用的用户提供稳定性和可预测性。

    ## 发布节奏

    我们预计**主版本**发布间隔至少 6-12 个月，以确保生产应用的稳定性。

    **次版本**通常每 1-2 个月发布一次，包含新功能和改进。

    **修订版**根据需要发布，通常每周一次，用于解决错误和安全问题。

    ## API 稳定性

    ### 稳定 API

    所有没有特殊前缀的 API 均被视为稳定的，可在生产环境中使用。我们在主版本内保持稳定功能的向后兼容性。

    ### Beta 功能

    文档中标记为 `beta` 的功能：

    - 功能已完整并已测试
    - 可以在生产环境中使用，但可能会发生变化
    - 可能根据用户反馈进行 API 微调

    ### 实验性功能

    标记为 `experimental` 或 `alpha` 的功能：

    - 正在积极开发中
    - 可能会发生重大变化或被移除
    - 在生产环境中使用时请谨慎

    ### 内部 API

    以下划线（`_`）开头或明确标记为内部的 API：

    - 不属于公共 API
    - 可能会在无通知的情况下变更
    - 不应直接使用

    ## 废弃策略

    当废弃功能时：

    1. **废弃通知**：功能会被标记为废弃，并提供清晰的迁移指南
    2. **过渡期**：已废弃功能至少保留一个次版本的正常使用期
    3. **移除**：功能仅在主版本发布时移除
    4. **迁移支持**：我们提供迁移指南，并在可能时提供自动化工具

    ## 平台兼容性

    :::python
    ### Python 支持

    - 我们支持 Python 软件基金会积极维护的 Python 版本
    - Python 版本要求只会在主版本发布时变更
    - 当前要求 Python 3.10 及以上
    :::

    :::js
    ### JavaScript/TypeScript 支持

    - 我们支持 Node.js LTS 版本
    - 所有公共 API 均提供 TypeScript 类型定义
    - 客户端组件的浏览器兼容性已在文档中说明
    :::

    ## 破坏性变更

    破坏性变更仅在主版本中引入，包括：

    - 移除已废弃的 API
    - 更改必需参数
    - 更改影响现有应用的默认行为
    - 更新最低 Python/Node.js 版本要求

    ## 迁移支持

    对于主版本升级，我们提供：

    - 全面的迁移指南
    - 可行时提供自动化迁移脚本
    - 对前一个主版本的延长支持期
    - 所有破坏性变更的清晰文档说明

    ## 长期支持（LTS）

    LangGraph 遵循长期支持（LTS）策略，为生产应用提供稳定性保障：

    ### 发布状态定义

    - **ACTIVE**：当前活跃开发中，包含错误修复、安全补丁和新功能

    ### 当前 LTS 版本

    - **状态**：ACTIVE，直到 2.0 发布
    - **支持期限**：2.0 发布后，1.0 将进入 MAINTENANCE 模式，至少维持 1 年
    - **语义化版本控制匹配**：用户可以在次版本之间安全升级（例如，从 1.0 到 1.1）而不会遇到破坏性变更

    ### 旧版本支持

    <Tab title="Deep Agents">
    :::python
    Deep Agents（`deepagents`）是一个用于构建高级 Agent 架构的快速迭代包。

    ## 预发布（Pre-1.0）状态

    Deep Agents 目前处于预发布开发阶段。虽然我们尽量减少破坏性变更，但随着我们从社区中吸取经验并适应快速变化的 Agent 应用领域，API 可能会发生演进。

    ## 发布节奏

    **次版本**发布（例如，`0.1.0` 到 `0.2.0`）可能包含新功能以及潜在的破坏性变更。

    **修订版**发布（例如，`0.1.0` 到 `0.1.1`）包含错误修复和小幅改进，不包含破坏性变更。

    ## API 稳定性

    作为预发布包：

    - API 可能在次版本之间发生变化
    - 当发生变更时，我们会尽量减少干扰并提供清晰的迁移指南
    - 标记为 `experimental` 或 `alpha` 的功能可能会有更大的变化

    ## 废弃策略

    当废弃 Deep Agents 中的功能时：

    - 我们会尽可能提前发出废弃警告
    - 迁移指南会包含在发布说明中
    - 已废弃的功能会在后续的次版本发布中移除

    ## 迈向 1.0 之路

    Deep Agents 将在以下条件满足时达到 1.0：

    - 核心 API 已根据社区反馈趋于稳定
    - 该包已在生产环境中经受充分验证
    - 核心功能不再预期会有破坏性变更

    1.0 发布后，Deep Agents 将采用与 LangChain 和 LangGraph 相同的语义化版本控制和 LTS 策略。
