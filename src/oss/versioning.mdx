我们的开源版本号遵循 [Semantic Versioning](https://semver.org/)  规范，格式为：`主版本号.次版本号.修订号`。

- **主版本号（Major）**：包含破坏性 API 更新，需要修改代码。
- **次版本号（Minor）**：新增功能和改进，保持向后兼容。
- **修订号（Patch）**：错误修复和小幅改进。

例如：

- `1.0.0`：首个稳定版本，API 已达到生产就绪状态
- `1.1.0`：以向后兼容的方式新增功能
- `1.0.1`：向后兼容的错误修复

## API 稳定性

我们通过以下方式标识 API 的稳定性：

### 稳定 API

所有没有特殊前缀的 API 均被视为稳定的，可以在生产环境中使用。我们保证稳定功能的向后兼容性，仅在主版本号升级时引入破坏性变更。

### Beta API

标记为 `beta` 的 API 功能已完整，但可能会根据用户反馈进行微调。它们可以安全地用于生产环境，但未来可能需要小幅调整。

### Alpha API

标记为 `alpha` 的 API 属于实验性功能，可能会发生较大变化。在生产环境中使用时请谨慎。

### 已废弃 API

标记为 `deprecated` 的 API 将在未来的主版本中移除。我们会尽可能指明计划移除的版本。处理废弃 API 的建议：

1. 切换到推荐的替代 API
2. 参照迁移指南（随主版本发布）
3. 可用时使用自动化迁移工具

### 内部 API

某些 API 被明确标记为“内部”，主要有以下两种方式：

- 文档中明确提到为内部实现的内容。如果文档说明某内容是内部的，它可能会发生变化。
- 以下划线（**`_`**）开头的函数、方法和其他对象。这是 Python 表示私有成员的标准约定；任何以单个 **`_`** 开头的方法都属于内部 API。
    - **例外：**某些方法以 `_` 开头但未包含实现，它们旨在由子类覆写并提供实现。这类方法通常属于 LangChain 的**公共 API**。

## 发布周期

<AccordionGroup>
    <Accordion title="主版本发布">
        主版本发布（例如，`1.0.0` → `2.0.0`）可能包括：

        - 破坏性 API 变更
        - 移除已废弃的功能
        - 重大架构改进

        我们提供：

        - 详细的迁移指南
        - 可用时提供自动化迁移工具
        - 对前一个主版本的延长支持期
    </Accordion>

    <Accordion title="次版本发布">
        次版本发布（例如，`1.0.0` → `1.1.0`）包括：

        - 新功能
        - 性能改进
        - 新增可选参数
        - 向后兼容的增强
    </Accordion>

    <Accordion title="修订版发布">
        修订版发布（例如，`1.0.0` → `1.0.1`）包括：

        - 错误修复
        - 安全更新
        - 文档改进
        - 不涉及 API 变更的性能优化
    </Accordion>
</AccordionGroup>

## 版本支持策略

- **最新主版本**：全面支持，持续活跃开发（ACTIVE 状态）
- **前一个主版本**：在下一个主版本发布后的 12 个月内提供安全更新和关键错误修复（MAINTENANCE 状态）
- **更早版本**：仅提供社区支持

### 长期支持（LTS）版本

LangChain 和 LangGraph 1.0 均被指定为 LTS 版本：

- 1.0 版本将保持 ACTIVE 状态，直到 2.0 版本发布
- 2.0 版本发布后，1.0 版本将进入 MAINTENANCE 模式，至少维护 1 年
- LTS 版本遵循语义化版本控制（semver），可以安全地在次版本之间升级
- 旧版本（LangChain 0.3 和 LangGraph 0.4）处于 MAINTENANCE 模式，维护至 2026 年 12 月

### 预发布（Pre-1.0）包

:::python
**Deep Agents**（`deepagents`）是一个处于活跃开发中的预发布包。作为一个快速迭代的包，其 API 可能在次版本之间发生变化，但我们会尽量减少破坏性变更。Deep Agents 在达到 1.0 版本后将采用与 LangChain 和 LangGraph 相同的 LTS 策略。
:::

更多关于发布状态和支持时间线的详细信息，请参见[发布策略](/oss/release-policy)。

## 升级

:::python
<CodeGroup>

```bash LangChain
# Upgrade to the latest version
pip install -U langchain-core langchain

# Upgrade to a specific version
pip install langchain-core==1.0.0
```

```bash LangGraph
# Upgrade to the latest version
pip install -U langgraph

# Upgrade to a specific version
pip install langgraph==1.0.0
```

```bash Deep Agents
# Upgrade to the latest version
pip install -U deepagents

# Upgrade to a specific version
pip install deepagents==0.1.0
```
</CodeGroup>
:::

:::js
<CodeGroup>

```bash LangChain
# Upgrade to the latest version
npm update langchain @langchain/core

# Install a specific version
npm install langchain@1.0.0 @langchain/core@1.0.0
```

```bash LangGraph
# Upgrade to the latest version
npm update @langchain/langgraph

# Install a specific version
npm install @langchain/langgraph@1.0.0
```
</CodeGroup>
:::

## 预发布版本

我们会不定期发布 alpha 和 beta 版本供早期测试：

- **Alpha**（例如，`1.0.0a1`）：早期预览，预计会有较大变化
- **Beta**（例如，`1.0.0b1`）：功能已完整，可能有小幅调整
- **候选发布版**（例如，`1.0.0rc1`）：稳定版发布前的最终测试
