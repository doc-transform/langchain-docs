<Tip>
**兼容性**

仅在 Node.js 环境中可用。
</Tip>

本模块基于 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) Node.js 绑定，用于 [llama.cpp](https://github.com/ggerganov/llama.cpp)，允许你与本地运行的 LLM 交互。这让你可以使用更小的量化模型，能够在笔记本环境中运行，非常适合测试和原型验证而无需产生费用！

## 设置

你需要安装 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) 模块的主版本 `3` 来与本地模型通信。

```bash npm
npm install -S node-llama-cpp@3
```
<Tip>
See [this section for general instructions on installing LangChain packages](/oss/langchain/install).
</Tip>

```bash npm
npm install @langchain/community @langchain/core
```
你还需要一个本地的 Llama 3 模型（或 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) 支持的模型）。你需要将模型的路径作为参数传递给 LlamaCpp 模块（见示例）。

开箱即用的 `node-llama-cpp` 针对 MacOS 平台进行了优化，支持 Apple M 系列处理器的 Metal GPU。如果你需要关闭该功能或需要支持 CUDA 架构，请参见 [node-llama-cpp](https://withcatai.github.io/node-llama-cpp/) 文档。

LangChain.js 贡献者注意：如果你想运行与此模块相关的测试，你需要将本地模型的路径放在环境变量 `LLAMA_PATH` 中。

## Llama3 安装指南

在本地运行 Llama3 模型是前置条件，以下是获取和构建 Llama 3.1-8B（最小版本）然后量化以便在笔记本上舒适运行的快速指南。为此你需要机器上有 `python3`（建议 3.11），以及 `gcc` 和 `make` 以便构建 `llama.cpp`。

### 获取 Llama3 模型

要获取 Llama3，你需要访问 [Meta AI](https://ai.meta.com/resources/models-and-libraries/llama-downloads/) 并申请访问权限。Meta AI 批准你的访问后，你将收到一封包含唯一 URL 的电子邮件用于访问文件，下一步将需要用到。
现在创建一个工作目录，例如：

```
mkdir llama3
cd llama3
```
现在我们需要前往 Meta AI 的 `llama-models` 仓库，可以在[这里](https://github.com/meta-llama/llama-models)找到。仓库中有下载你选择的模型的说明，你需要使用邮件中收到的唯一 URL。
本教程剩余部分假设你已下载了 `Llama3.1-8B`，但从现在起任何模型都应该可以工作。下载模型后，确保保存模型下载路径，稍后会用到。

### 转换和量化模型

在此步骤中我们需要使用 `llama.cpp`，所以需要下载该仓库。

```
cd ..
git clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp
```
现在我们需要构建 `llama.cpp` 工具并设置 `python` 环境。在以下步骤中，假设你的 Python 可以通过 `python3` 运行，虚拟环境命名为 `llama3`，请根据自己的情况进行调整。

```
cmake -B build
cmake --build build --config Release
python3 -m venv llama3
source llama3/bin/activate
```
激活 llama3 环境后，你应该看到 `(llama3)` 前缀在命令提示符前面，表示这是当前活动环境。注意：如果你需要回来构建另一个模型或重新量化模型，不要忘记再次激活环境，同时如果你更新了 `llama.cpp`，你可能需要重新构建工具并安装新的或更新的依赖！现在我们有了活动的 Python 环境，需要安装 Python 依赖。

```
python3 -m pip install -r requirements.txt
```
完成后，我们可以开始转换和量化 Llama3 模型以便通过 `llama.cpp` 在本地使用。需要先转换为 Hugging Face 模型，然后转换为 GGUF 模型。
首先，我们需要找到脚本 `convert_llama_weights_to_hf.py`。将此脚本复制到当前工作目录。注意，使用该脚本可能需要 pip 安装额外的依赖，根据需要安装即可。
然后，我们需要转换模型，转换之前先创建目录来存储 Hugging Face 转换结果和最终模型。

```
mkdir models/8B
mkdir models/8B-GGUF
python3 convert_llama_weights_to_hf.py --model_size 8B --input_dir <dir-to-your-model> --output_dir models/8B --llama_version 3
python3 convert_hf_to_gguf.py --outtype f16 --outfile models/8B-GGUF/gguf-llama3-f16.bin models/8B
```
这应该会在我们创建的目录中生成转换后的 Hugging Face 模型和最终的 GGUF 模型。注意，这只是转换后的模型，大约 16GB，下一步我们将它量化到约 4GB。

```
./build/bin/llama-quantize ./models/8B-GGUF/gguf-llama3-f16.bin ./models/8B-GGUF/gguf-llama3-Q4_0.bin Q4_0
```
运行后应该会在 `models\8B-GGUF` 目录中创建一个新模型 `gguf-llama3-Q4_0.bin`，这就是我们可以在 langchain 中使用的模型。你可以使用 `llama.cpp` 工具来验证模型是否正常工作。

```
./build/bin/llama-cli -m ./models/8B-GGUF/gguf-llama3-Q4_0.bin -cnv -p "You are a helpful assistant"
```

运行此命令会启动模型进行聊天会话。顺便说一下，如果你磁盘空间不足，这个小模型是我们唯一需要的，你可以备份和/或删除原始和转换后的 13.5GB 模型。

## 用法

```typescript
import { LlamaCpp } from "@langchain/community/llms/llama_cpp";

const llamaPath = "/Replace/with/path/to/your/model/gguf-llama3-Q4_0.bin";
const question = "Where do Llamas come from?";

const model = await LlamaCpp.initialize({ modelPath: llamaPath });

console.log(`You: ${question}`);
const response = await model.invoke(question);
console.log(`AI : ${response}`);
```

## 流式输出

```typescript
import { LlamaCpp } from "@langchain/community/llms/llama_cpp";

const llamaPath = "/Replace/with/path/to/your/model/gguf-llama3-Q4_0.bin";

const model = await LlamaCpp.initialize({
  modelPath: llamaPath,
  temperature: 0.7,
});

const prompt = "Tell me a short story about a happy Llama.";

const stream = await model.stream(prompt);

for await (const chunk of stream) {
  console.log(chunk);
}

/*


 Once
  upon
  a
  time
 ,
  in
  the
  rolling
  hills
  of
  Peru
 ...
 */
```;
