## 安装和配置

要使用此功能，请安装 `@sap/hana-langchain` 包及其依赖：

```bash
npm install @sap/hana-langchain @langchain/core@latest @langchain/classic@latest langchain@latest
```

创建与 SAP HANA Cloud 实例的连接。

```typescript
import * as dotenv from 'dotenv';
// 如需加载环境变量
dotenv.config();

import hanaClient from "@sap/hana-client";

const connectionParams = {
  host: process.env.HANA_DB_ADDRESS,
  port: process.env.HANA_DB_PORT,
  user: process.env.HANA_DB_USER,
  password: process.env.HANA_DB_PASSWORD,
};
const client = hanaClient.createConnection(connectionParams);

// 连接到 hanaDB
await new Promise<void>((resolve, reject) => {
  client.connect((err: Error) => {
    // 此处使用箭头函数
    if (err) {
      reject(err);
    } else {
      console.log("Connected to SAP HANA successfully.");
      resolve();
    }
  });
});
```

`HanaSparqlQAChain` 将以下几个部分结合在一起：

1. **感知 Schema 的 SPARQL 生成**
2. **对 SAP HANA 执行查询**
3. **自然语言答案格式化**

## 初始化

* 一个 **LLM** 用于生成和解释查询
* 一个 **`HanaRdfGraph`**（包含连接、`graph_uri` 和本体）

按照[此处步骤](/oss/integrations/graphs/sap_hana_rdf_graph)了解如何创建 `HanaRdfGraph` 实例。

导入 `HanaSparqlQAChain`

```typescript
import { HanaSparqlQAChain } from "@sap/hana-langchain";
const qaChain = HanaSparqlQAChain.fromLLM({ llm, graph, allowDangerousRequests: true, verbose: true });
```

## \u流水线概览

1. SPARQL 生成
   - 使用 SPARQL 生成提示词
   - 输入：`schema`（来自 `graph.getSchema()` 的 Turtle 格式）、`prompt`（用户问题）
2. 查询后处理
   - 从模型输出中提取 SPARQL
   - 如缺少则注入 `FROM <graph_uri>`
   - 确保包含常用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）
3. 执行
   - `graph.query(generatedSparql)`
4. 答案生成
   - 使用 QA 提示词，输入：`context`（结果）、`prompt`（原始问题）

## 提示词模板

### “SPARQL 生成”提示词

`sparqlGenerationPrompt` 用于引导 LLM 根据用户问题和提供的 schema 生成 SPARQL 查询。

### 回答提示词

`qaPrompt` 指示 LLM 仅根据数据库结果创建自然语言答案。
默认提示词可在此处找到：[`prompts.ts`](https://github.com/SAP/langchainjs-integration-for-sap-hana-cloud/blob/main/src/chains/graph_qa/prompts.ts)

## 自定义提示词

你可以在初始化时覆盖默认值：

```typescript
const qaChain = HanaSparqlQAChain.fromLLM({
  llm,
  graph,
  allowDangerousRequests: true,
  verbose: true,
  sparqlGenerationPrompt: YOUR_SPARQL_PROMPT,
  qaPrompt: YOUR_QA_PROMPT,
});
```

> - `sparql_generation_prompt` must have the input variables: `["schema", "prompt"]`
> - `qa_prompt` must have the input variables: `["context", "prompt"]`

## 示例：在“电影”知识图谱上进行问答

**前提条件**：
你必须拥有一个启用了 **triple store** 功能的 SAP HANA Cloud 实例。
详细说明请参考：[启用 Triple Store](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/enable-triple-store/)<br />
加载 `kgdocu_movies` 示例数据。参见[知识图谱示例](https://help.sap.com/docs/hana-cloud-database/sap-hana-cloud-sap-hana-database-knowledge-graph-guide/knowledge-graph-example)。

下面我们将：

1. 创建指向“电影”数据图的 `HanaRdfGraph` 实例
2. 将其封装在由 LLM 驱动的 `HanaSparqlQAChain` 中
3. 提出自然语言问题并打印链式调用的响应

这演示了 LLM 如何在幕后生成 SPARQL、对 SAP HANA 执行查询并返回人类可读的答案。

## 底层运作原理

1. **SPARQL 生成**
  链式调用使用你的 Turtle 格式本体（`graph.getSchema()`）和用户问题，通过 `SPARQL_GENERATION_SELECT_PROMPT` 调用 LLM。LLM 然后生成一个针对你的 schema 定制的有效 `SELECT` 查询。
2. **预处理与执行**
  * **提取与清理**：从 LLM 响应中提取原始 SPARQL 文本。
  * **注入图上下文**：如缺少则添加 `FROM <graph_uri>`，并确保声明了常用前缀（`rdf:`、`rdfs:`、`owl:`、`xsd:`）。
  * **在 HANA 上执行**：通过 `HanaRdfGraph.query()` 在你的命名图上执行最终查询。
3. **答案生成**
  返回的 CSV（或 Turtle）结果再次输入 LLM——这次使用 `SPARQL_QA_PROMPT`。LLM 仅根据检索到的数据生成简洁、人类可读的答案，不会产生幻觉。
