**兼容性**：仅在 Node.js 上可用。
</Tip>

Google Vertex AI Matching Engine “提供业界领先的高规模低延迟向量数据库。这些向量数据库通常被称为向量相似性匹配或近似最近邻（ANN）服务”。

## 设置

<Warning>
**本模块需要一个已创建的端点和已部署的索引，因为**

创建时间接近一个小时。要了解更多信息，请参见 LangChain Python 文档 [创建索引并将其部署到端点](https://python.langchain.com/docs/integrations/vectorstores/google_vertex_ai_vector_search/#deploy-index-to-the-endpoint)。
</Warning>

在运行此代码之前，你应该确保在 Google Cloud 控制台中为相关项目启用了 Vertex AI API，并且你已通过以下方法之一进行了 Google Cloud 身份验证：

- 你已登录到允许访问该项目的账户（使用 `gcloud auth application-default login`）。
- 你正在使用允许访问该项目的服务账户的机器上运行。
- 你已下载了允许访问该项目的服务账户的凭据，并将 `GOOGLE_APPLICATION_CREDENTIALS` 环境变量设置为此文件的路径。

安装身份验证库：

```bash npm
npm install @langchain/community @langchain/core google-auth-library
```
Matching Engine 不存储实际的文档内容，只存储嵌入。因此，你需要一个文档存储。下面的示例使用 Google Cloud Storage，需要以下包：

```bash npm
npm install @google-cloud/storage
```
## 用法

### 初始化引擎

创建 `MatchingEngine` 对象时，你需要一些关于 Matching Engine 配置的信息。你可以从 Matching Engine 的 Cloud Console 获取这些信息：

- Index 的 ID
- Index Endpoint 的 ID

你还需要一个文档存储。虽然 `InMemoryDocstore` 适用于初始测试，但你将需要使用类似 [GoogleCloudStorageDocstore](https://api.js.langchain.com/classes/_langchain_community.stores_doc_gcs.GoogleCloudStorageDocstore.html) 的工具来更持久地存储。

```typescript
import { MatchingEngine } from "@langchain/community/vectorstores/googlevertexai";
import { Document } from "@langchain/classic/document";
import { SyntheticEmbeddings } from "@langchain/classic/embeddings/fake";
import { GoogleCloudStorageDocstore } from "@langchain/community/stores/doc/gcs";

const embeddings = new SyntheticEmbeddings({
  vectorSize: Number.parseInt(
    process.env.SYNTHETIC_EMBEDDINGS_VECTOR_SIZE ?? "768",
    10
  ),
});

const store = new GoogleCloudStorageDocstore({
  bucket: process.env.GOOGLE_CLOUD_STORAGE_BUCKET!,
});

const config = {
  index: process.env.GOOGLE_VERTEXAI_MATCHINGENGINE_INDEX!,
  indexEndpoint: process.env.GOOGLE_VERTEXAI_MATCHINGENGINE_INDEXENDPOINT!,
  apiVersion: "v1beta1",
  docstore: store,
};

const engine = new MatchingEngine(embeddings, config);
```
### 添加文档

```typescript
const doc = new Document({ pageContent: "this" });
await engine.addDocuments([doc]);
```
文档中的任何元数据都会被转换为 Matching Engine 的 "允许列表" 值，可用于在查询时进行过滤。

```typescript
const documents = [
  new Document({
    pageContent: "this apple",
    metadata: {
      color: "red",
      category: "edible",
    },
  }),
  new Document({
    pageContent: "this blueberry",
    metadata: {
      color: "blue",
      category: "edible",
    },
  }),
  new Document({
    pageContent: "this firetruck",
    metadata: {
      color: "red",
      category: "machine",
    },
  }),
];

// 添加所有文档
await engine.addDocuments(documents);
```
文档假定也有一个可用的 "id" 参数。如果未设置，则会分配一个 ID 并作为 Document 的一部分返回。

### 查询文档

使用任何标准方法执行直接的 k-最近邻搜索，返回所有结果：

```typescript
const results = await engine.similaritySearch("this");
```
### 使用过滤器/限制条件查询文档

我们可以根据为文档设置的元数据来限制返回的文档。所以如果我们只想将结果限制为红色的项目，可以这样做：

```typescript
import { Restriction } from `@langchain/community/vectorstores/googlevertexai`;

const redFilter: Restriction[] = [
  {
    namespace: "color",
    allowList: ["red"],
  },
];
const redResults = await engine.similaritySearch("this", 4, redFilter);
```
如果我们想做更复杂的操作，比如是红色但不可食用的项目：

```typescript
const filter: Restriction[] = [
  {
    namespace: "color",
    allowList: ["red"],
  },
  {
    namespace: "category",
    denyList: ["edible"],
  },
];
const results = await engine.similaritySearch("this", 4, filter);
```
### 删除文档

删除文档是通过 ID 完成的。

```typescript
import { IdDocument } from `@langchain/community/vectorstores/googlevertexai`;

const oldResults: IdDocument[] = await engine.similaritySearch("this", 10);
const oldIds = oldResults.map( doc => doc.id! );
await engine.delete({ids: oldIds});
```
