<Tip>
**兼容性**

仅在 Node.js 环境中可用。
</Tip>

本模块基于 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) Node.js 绑定，用于 [llama.cpp](https://github.com/ggerganov/llama.cpp)，允许你与本地运行的 LLM 交互。这让你可以使用更小的量化模型，能够在笔记本环境中运行，非常适合测试和原型验证而无需产生费用！

## 设置

你需要安装 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) 模块的主版本 `3` 来与本地模型通信。

```bash npm
npm install -S node-llama-cpp@3
```
<Tip>
See [this section for general instructions on installing LangChain packages](/oss/langchain/install).
</Tip>

```bash npm
npm install @langchain/community @langchain/core
```

你还需要一个本地的 Llama 3 模型（或 [node-llama-cpp](https://github.com/withcatai/node-llama-cpp) 支持的模型）。你需要将模型的路径作为参数传递给 LlamaCpp 模块（见示例）。

开箱即用的 `node-llama-cpp` 针对 MacOS 平台进行了优化，支持 Apple M 系列处理器的 Metal GPU。如果你需要关闭该功能或需要支持 CUDA 架构，请参见 [node-llama-cpp](https://withcatai.github.io/node-llama-cpp/) 文档。

关于获取和准备 `llama3` 的建议，请参见本模块的 LLM 版本文档。

LangChain.js 贡献者注意：如果你想运行与此模块相关的测试，你需要将本地模型的路径放在环境变量 `LLAMA_PATH` 中。

## 用法

### 基本用法

我们需要提供本地 Llama3 模型的路径，同时本模块中的 `embeddings` 属性始终设置为 `true`。

```typescript
import { LlamaCppEmbeddings } from "@langchain/community/embeddings/llama_cpp";

const llamaPath = "/Replace/with/path/to/your/model/gguf-llama3-Q4_0.bin";

const embeddings = await LlamaCppEmbeddings.initialize({
  modelPath: llamaPath,
});

const res = embeddings.embedQuery("Hello Llama!");

console.log(res);

/*
	[ 15043, 365, 29880, 3304, 29991 ]
*/
```

### 文档嵌入

```typescript
import { LlamaCppEmbeddings } from "@langchain/community/embeddings/llama_cpp";

const llamaPath = "/Replace/with/path/to/your/model/gguf-llama3-Q4_0.bin";

const documents = ["Hello World!", "Bye Bye!"];

const embeddings = await LlamaCppEmbeddings.initialize({
  modelPath: llamaPath,
});

const res = await embeddings.embedDocuments(documents);

console.log(res);

/*
	[ [ 15043, 2787, 29991 ], [ 2648, 29872, 2648, 29872, 29991 ] ]
*/
```
