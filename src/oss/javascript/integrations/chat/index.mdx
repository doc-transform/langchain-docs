[对话模型](/oss/langchain/models)是以[消息](/oss/langchain/messages)序列作为输入并返回消息作为输出的语言模型<Tooltip tip="不遵循对话模型接口的旧版模型，使用以字符串为输入并返回字符串为输出的接口。这类模型的名称中通常不包含 'Chat' 前缀，或包含 'LLM' 后缀。">（与纯文本模型相对）</Tooltip>。

## 安装和使用

<Tip>
    安装 LangChain 包的通用说明请参见[此章节](/oss/langchain/install)。
</Tip>

<AccordionGroup>
    <Accordion title="OpenAI">
        安装:

        <CodeGroup>
            ```bash npm
            npm i @langchain/openai
            ```
            ```bash yarn
            yarn add @langchain/openai
            ```
            ```bash pnpm
            pnpm add @langchain/openai
            ```
        </CodeGroup>

        添加环境变量:

        ```bash
        OPENAI_API_KEY=your-api-key
        ```

        实例化模型:

        ```typescript
        import { ChatOpenAI } from "@langchain/openai";

        const model = new ChatOpenAI({ model: "gpt-4.1-mini" });
        ```
        ```javascript
        await model.invoke("Hello, world!")
        ```
    </Accordion>

## Chat Completions API

某些模型提供商提供了与 OpenAI（旧版）[Chat Completions API](https://platform.openai.com/docs/guides/completions) 兼容的端点。在这种情况下，你可以使用 [`ChatOpenAI`](/oss/integrations/chat/openai) 配合自定义的 `base_url` 来连接这些端点。请注意，基于 Chat Completions API 构建的功能可能不会被 `ChatOpenAI` 完全支持；在这种情况下，建议使用提供商专用的类（如果可用的话）。

<Accordion title="示例：OpenRouter">
    要使用 OpenRouter，你需要注册一个账户并获取 [API 密钥](https://openrouter.ai/docs/api/reference/authentication)。

    ```typescript
    import { ChatOpenAI } from "@langchain/openai";

    const model = new ChatOpenAI({
        model: "...", // Specify a model available on OpenRouter
        configuration: {
        apiKey: "OPENROUTER_API_KEY",
        baseURL: "https://openrouter.ai/api/v1",
        }
    });
    ```

    更多详情请参阅 [OpenRouter 文档](https://openrouter.ai/docs/quickstart)。
</Accordion>
