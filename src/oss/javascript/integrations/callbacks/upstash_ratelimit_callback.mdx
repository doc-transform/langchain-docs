---

在本指南中，我们将介绍如何使用 `UpstashRatelimitHandler` 根据请求数或 token 数添加速率限制。该处理器使用 [Upstash 的速率限制库](https://github.com/upstash/ratelimit-js/)，它依赖 [Upstash Redis](https://upstash.com/docs/redis/overall/getstarted)。

Upstash Ratelimit 的工作原理是每次调用 `limit` 方法时向 Upstash Redis 发送 HTTP 请求。系统会检查并更新用户的剩余 token/请求数。根据剩余 token 数，我们可以阻止执行高成本操作，如调用 LLM 或查询向量存储：

```tsx
const response = await ratelimit.limit();
if (response.success) {
  execute_costly_operation();
}
```

`UpstashRatelimitHandler` 可以让你在几分钟内将这种速率限制逻辑集成到你的链式调用中。

## 安装

首先，你需要前往 [Upstash 控制台](https://console.upstash.com/login)并创建一个 Redis 数据库（[参见文档](https://upstash.com/docs/redis/overall/getstarted)）。创建数据库后，你需要设置环境变量：

```
UPSTASH_REDIS_REST_URL="****"
UPSTASH_REDIS_REST_TOKEN="****"
```

接下来，你需要安装 Upstash Ratelimit 和 `@langchain/community`：

<Tip>
安装 LangChain 包的通用说明请参见[此章节](/oss/langchain/install)。
</Tip>

```bash npm
npm install @upstash/ratelimit @langchain/community @langchain/core
```
现在你可以为你的链式调用添加速率限制了！

## 按请求限速

假设我们希望允许用户每分钟调用链式调用 10 次。实现方法非常简单：

```tsx
const UPSTASH_REDIS_REST_URL = "****";
const UPSTASH_REDIS_REST_TOKEN = "****";

import {
  UpstashRatelimitHandler,
  UpstashRatelimitError,
} from "@langchain/community/callbacks/handlers/upstash_ratelimit";
import { RunnableLambda } from "@langchain/core/runnables";
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

// 创建速率限制
const ratelimit = new Ratelimit({
  redis: new Redis({
    url: UPSTASH_REDIS_REST_URL,
    token: UPSTASH_REDIS_REST_TOKEN,
  }),
  // 每个窗口期 10 个请求，窗口大小为 60 秒：
  limiter: Ratelimit.fixedWindow(10, "60 s"),
});

// 创建处理器
const user_id = "user_id"; // 应为获取用户 ID 的方法
const handler = new UpstashRatelimitHandler(user_id, {
  requestRatelimit: ratelimit,
});

// 创建模拟链式调用
const chain = new RunnableLambda({ func: (str: string): string => str });

try {
  const response = await chain.invoke("hello world", {
    callbacks: [handler],
  });
  console.log(response);
} catch (err) {
  if (err instanceof UpstashRatelimitError) {
    console.log("Handling ratelimit.");
  }
}
```
请注意，我们是在 `invoke` 方法中传递处理器，而不是在定义链式调用时传递。

对于 `FixedWindow` 之外的速率限制算法，请参见 [upstash-ratelimit 文档](https://upstash.com/docs/oss/sdks/ts/ratelimit/algorithms)。

在执行流水线中的任何步骤之前，速率限制会检查用户是否已超过请求限制。如果超过，将抛出 `UpstashRatelimitError`。

## 按 token 限速

另一种选择是根据以下指标对链式调用进行速率限制：

1. 提示词中的 token 数
2. 提示词和 LLM 完成中的 token 总数

此功能仅在链式调用中包含 LLM 时有效。另一个要求是你使用的 LLM 应在其 `LLMOutput` 中返回 token 用量。返回的 token 用量字典格式取决于具体的 LLM。要了解如何根据 LLM 配置处理器，请参见下面配置部分的末尾。

### 工作原理

处理器会在调用 LLM 之前获取剩余 token 数。如果剩余 token 大于 0，则调用 LLM；否则抛出 `UpstashRatelimitError`。

LLM 调用完成后，token 用量信息将从用户的剩余 token 中扣除。在链式调用的这个阶段不会抛出错误。

### 配置

对于第一种配置，只需这样初始化处理器：

```tsx
const user_id = "user_id"; // 应为获取用户 ID 的方法
const handler = new UpstashRatelimitHandler(user_id, {
  requestRatelimit: ratelimit,
});
```
对于第二种配置，这样初始化处理器：

```tsx
const user_id = "user_id"; // 应为获取用户 ID 的方法
const handler = new UpstashRatelimitHandler(user_id, {
  tokenRatelimit: ratelimit,
});
```
你也可以同时使用基于请求和基于 token 的速率限制，只需同时传入 `request_ratelimit` 和 `token_ratelimit` 参数即可。

要使 token 用量统计正常工作，LangChain.js 中的 LLM 步骤应以以下格式返回 token 用量字段：

```json
{
  "tokenUsage": {
    "totalTokens": 123,
    "promptTokens": 456,
    "otherFields: "..."
  },
  "otherFields: "..."
}
```
但并非 LangChain.js 中的所有 LLM 都遵循此格式。如果你的 LLM 使用不同的键名返回相同的值，可以使用 `llmOutputTokenUsageField`、`llmOutputTotalTokenField` 和 `llmOutputPromptTokenField` 参数传递给处理器：

```tsx
const handler = new UpstashRatelimitHandler(
  user_id,
  {
    requestRatelimit: ratelimit
    llmOutputTokenUsageField: "usage",
    llmOutputTotalTokenField: "total",
    llmOutputPromptTokenField: "prompt"
  }
)
```
以下是一个使用 LLM 的链式调用示例：

```tsx
const UPSTASH_REDIS_REST_URL = "****";
const UPSTASH_REDIS_REST_TOKEN = "****";
const OPENAI_API_KEY = "****";

import {
  UpstashRatelimitHandler,
  UpstashRatelimitError,
} from "@langchain/community/callbacks/handlers/upstash_ratelimit";
import { RunnableLambda, RunnableSequence } from "@langchain/core/runnables";
import { OpenAI } from "@langchain/openai";
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

// 创建速率限制
const ratelimit = new Ratelimit({
  redis: new Redis({
    url: UPSTASH_REDIS_REST_URL,
    token: UPSTASH_REDIS_REST_TOKEN,
  }),
  // 每个窗口期 500 个 token，窗口大小为 60 秒：
  limiter: Ratelimit.fixedWindow(500, "60 s"),
});

// 创建处理器
const user_id = "user_id"; // 应为获取用户 ID 的方法
const handler = new UpstashRatelimitHandler(user_id, {
  tokenRatelimit: ratelimit,
});

// 创建模拟链式调用
const asStr = new RunnableLambda({ func: (str: string): string => str });
const model = new OpenAI({
  apiKey: OPENAI_API_KEY,
});
const chain = RunnableSequence.from([asStr, model]);

// 使用处理器调用链式调用：
try {
  const response = await chain.invoke("hello world", {
    callbacks: [handler],
  });
  console.log(response);
} catch (err) {
  if (err instanceof UpstashRatelimitError) {
    console.log("Handling ratelimit.");
  }
}
```
