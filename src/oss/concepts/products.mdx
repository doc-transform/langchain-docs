## Agent 框架（如 LangChain）

Agent 框架提供了一系列抽象，使基于 LLM 的开发更容易上手。

[LangChain](/oss/langchain/overview) 是一个 Agent 框架，提供了结构化内容块、Agent 循环和中间件等抽象。

LangChain 的抽象设计兼顾了易用性和高级场景所需的灵活性。

虽然 LangChain 构建在 [LangGraph](/oss/langgraph/overview) 之上，但使用 LangChain 并不需要了解 LangGraph。

其他 Agent 框架还有 [Vercel's AI SDK](https://ai-sdk.dev/docs/introduction)、[CrewAI](https://www.crewai.com/)、[OpenAI Agents SDK](https://openai.github.io/openai-agents-python/)、[Google ADK](https://google.github.io/adk-docs/)、[LlamaIndex](https://www.llamaindex.ai/) 等。
### 何时使用 LangChain

在以下场景中推荐使用 LangChain：

- 需要快速构建 Agent 和自治应用。
- 需要模型、工具和 Agent 循环的标准化抽象。
- 希望使用简单易用且仍具灵活性的框架。
- 构建无需复杂编排的常规 Agent 应用。

## Agent 运行时（如 LangGraph）

Agent 运行时提供在生产环境中运行 Agent 所需的工具链，支持的能力包括：

- **持久化执行**：Agent 在故障后仍能保持状态，可长时间运行，并从中断处恢复。
- **流式输出**：支持工作流和响应的流式传输。
- **人机协作**：通过检查和修改 Agent 状态引入人工监督。
- **状态持久化**：支持线程级和跨线程的状态管理。
- **底层控制**：无需高层抽象，直接控制 Agent 编排逻辑。

[LangGraph](/oss/langgraph/overview) 是一个底层编排框架和运行时，用于构建、管理和部署长时运行的有状态 Agent。

Agent 框架通常更高层级，运行在 Agent 运行时之上。例如，LangChain 1.0 就构建在 LangGraph 之上。

其他 Agent 运行时还有 [Temporal](https://temporal.io/)、[Inngest](https://www.inngest.com/) 及其他持久化执行引擎。

### 何时使用 LangGraph

在以下场景中推荐使用 LangGraph：

- 需要对 Agent 编排进行底层精细控制。
- 需要为长时运行的有状态 Agent 提供持久化执行。
- 构建确定性步骤与 Agent 步骤相结合的复杂工作流。
- 需要生产级别的 Agent 部署基础设施。

## Agent 套件（如 Deep Agents SDK）

Agent 套件是一类开箱即用的 Agent 框架，内置了丰富的工具和能力，使构建复杂的长时运行 Agent 变得更加容易。支持的能力包括：

- **规划能力**：通过待办列表跟踪和管理多项任务。
- **任务委派**：通过子 Agent 委派工作，保持上下文清晰。
- **文件系统**：对不同可插拔存储后端上的文件进行读写访问。
- **Token 管理**：对话历史摘要和大型工具结果的自动清除。

[Deep Agents SDK](/oss/deepagents/overview) 构建在 LangGraph 之上，增加了规划能力、用于上下文管理的文件系统、生成子 Agent 的能力等。Deep Agents 专为需要规划和任务分解的复杂多步骤任务而设计。示例任务包括处理搜索结果、脚本以及状态中的其他产物。

其他 Agent 套件还有 [Claude Agent SDK](https://platform.claude.com/docs/en/agent-sdk/overview)、[Manus](https://manus.im/) 及其他编程 CLI 工具。

### 何时使用 Deep Agents SDK

在以下场景中推荐使用 [Deep Agents SDK](/oss/deepagents/overview)：

- 构建长时间运行的 Agent。
- 构建需要处理复杂多步骤任务的 Agent。
- 希望使用预置工具，如文件系统操作、Bash 执行和自动化上下文工程。
- 希望使用预置提示词和子 Agent。

## 功能对比

虽然使用 LangChain、LangGraph 和 Deep Agents 都能完成类似的任务，但它们的集成层级有所不同：

| ------- | --------- | ----------- | ----------- |
| Short-term memory | [Short-term memory](/oss/langchain/short-term-memory) | [Short-term memory](/oss/langgraph/add-memory#add-short-term-memory) | [`StateBackend`](/oss/deepagents/backends#statebackend-ephemeral) |
| Long-term memory | [Long-term memory](/oss/langchain/long-term-memory) | [Long-term memory](/oss/langgraph/add-memory#add-long-term-memory) | [Long-term memory](/oss/deepagents/long-term-memory) |
| Skills  | [Multi-agent skills](/oss/langchain/multi-agent/skills) | - | [Skills](/oss/deepagents/skills) |
| Subagents | [Multi-agent subagents](/oss/langchain/multi-agent/subagents) | [Subgraphs](/oss/langgraph/use-subgraphs) | [Subagents](/oss/deepagents/subagents) |
