LangGraph 受到塑造 Agent 未来的公司信赖——包括 Klarna、Replit、Elastic 等——它是一个低层编排框架和运行时，用于构建、管理和部署长时间运行的有状态 Agent。

LangGraph 非常低层，完全专注于 Agent 的**编排**。在使用 LangGraph 之前，我们建议你先熟悉一些构建 Agent 的组件，从[模型](/oss/langchain/models)和[工具](/oss/langchain/tools)开始。

我们在文档中常用 [LangChain](/oss/langchain/overview) 组件来集成模型和工具，但你不需要使用 LangChain 也能使用 LangGraph。如果你刚开始接触 Agent 或想要更高层的抽象，我们建议你使用 LangChain 的 [agents](/oss/langchain/agents)，它提供了常见 LLM 和工具调用循环的预构建架构。

LangGraph 专注于对 Agent 编排至关重要的底层能力：持久化执行、流式输出、人工干预等。

然后，创建一个简单的 hello world 示例：

:::python
```python
from langgraph.graph import StateGraph, MessagesState, START, END

def mock_llm(state: MessagesState):
    return {"messages": [{"role": "ai", "content": "hello world"}]}

graph = StateGraph(MessagesState)
graph.add_node(mock_llm)
graph.add_edge(START, "mock_llm")
graph.add_edge("mock_llm", END)
graph = graph.compile()

graph.invoke({"messages": [{"role": "user", "content": "hi!"}]})
```
:::

:::js
```typescript
import { StateSchema, MessagesValue, GraphNode, StateGraph, START, END } from "@langchain/langgraph";

const State = new StateSchema({
  messages: MessagesValue,
});

const mockLlm: GraphNode<typeof State> = (state) => {
  return { messages: [{ role: "ai", content: "hello world" }] };
};

const graph = new StateGraph(State)
  .addNode("mock_llm", mockLlm)
  .addEdge(START, "mock_llm")
  .addEdge("mock_llm", END)
  .compile();

await graph.invoke({ messages: [{ role: "user", content: "hi!" }] });
```
:::

## 核心优势

LangGraph 为*任何*长时间运行的有状态工作流或 Agent 提供低层支撑基础设施。LangGraph 不抽象提示词或架构，提供以下核心优势：

* [持久化执行](/oss/langgraph/durable-execution)：构建能够在故障中持久化并可长时间运行的 Agent，从中断处恢复执行。
* [人工干预](/oss/langgraph/interrupts)：通过在任意时刻检查和修改 Agent 状态来纳入人工监督。
* [全面的记忆](/oss/concepts/memory)：创建同时具备短期工作记忆（用于持续推理）和跨会话长期记忆的有状态 Agent。
* [使用 LangSmith 调试](/langsmith/home)：通过可视化工具深入了解复杂的 Agent 行为，跟踪执行路径、捕获状态转换并提供详细的运行时指标。
* [生产就绪的部署](/langsmith/deployments)：使用专为有状态、长时间运行工作流的独特挑战而设计的可扩展基础设施，自信地部署复杂的 Agent 系统。
