---
title: MULTIPLE_SUBGRAPHS
---

此错误发生在你在[节点内部调用子图](/oss/langgraph/use-subgraphs#call-a-subgraph-inside-a-node)多次，且子图使用 `checkpointer=True`（续传模式）编译时。

## 故障排除

根据你的需求选择以下方案之一：

:::python
1. **不需要中断？**使用 `checkpointer=False` 完全退出检查点机制：
   ```python
   subgraph = subgraph_builder.compile(checkpointer=False)
   ```

2. **需要中断但不需要跨调用持久化？**使用默认的继承模式，省略 `checkpointer`：
   ```python
   subgraph = subgraph_builder.compile()
   ```
   每次调用会获得唯一的命名空间，因此并行执行是有效的。子图每次都从头开始，但可以使用 `interrupt()`。

3. **需要跨调用持久化？**使用 `checkpointer=True`。LangGraph 会为每次调用分配基于位置的命名空间后缀（`calling_node`、`calling_node|1` 等）以防止冲突。要获得稳定的基于名称的命名空间，请用唯一的节点名称包装每个子图——参见[并行子图](/oss/langgraph/use-subgraphs#subgraph-persistence)。
:::

:::js
1. **不需要中断？**使用 `checkpointer: false` 完全退出检查点机制：
   ```typescript
   const subgraph = subgraphBuilder.compile({ checkpointer: false });
   ```

2. **需要中断但不需要跨调用持久化？**使用默认的继承模式，省略 `checkpointer`：
   ```typescript
   const subgraph = subgraphBuilder.compile();
   ```
   每次调用会获得唯一的命名空间，因此并行执行是有效的。子图每次都从头开始，但可以使用 `interrupt()`。

3. **需要跨调用持久化？**使用 `checkpointer: true`。LangGraph 会为每次调用分配基于位置的命名空间后缀（`calling_node`、`calling_node|1` 等）以防止冲突。要获得稳定的基于名称的命名空间，请用唯一的节点名称包装每个子图——参见[并行子图](/oss/langgraph/use-subgraphs#subgraph-persistence)。
:::
