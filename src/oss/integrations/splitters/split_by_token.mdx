语言模型有 Token 限制。你不应该超过 Token 限制。因此，当你将文本[分割](/oss/integrations/splitters/)成块时，计算 Token 数量是一个好主意。有许多种分词器。计算文本中的 Token 时，你应该使用与语言模型相同的分词器。

:::python
## tiktoken

<Note>
**[tiktoken](https://github.com/openai/tiktoken) 是 `OpenAI` 创建的一个快速 `BPE` 分词器。**
</Note>


我们可以使用 `tiktoken` 来估算 Token 使用量。对于 OpenAI 模型，它的计算可能更准确。

1. 文本如何分割：通过传入的字符。
2. 块大小如何计算：通过 `tiktoken` 分词器。

@[CharacterTextSplitter]、@[RecursiveCharacterTextSplitter] 和 @[TokenTextSplitter] 可以直接与 `tiktoken` 配合使用。


```python
pip install --upgrade --quiet langchain-text-splitters tiktoken
```


```python
from langchain_text_splitters import CharacterTextSplitter

# 这是一个可以被分割的长文档。
with open("state_of_the_union.txt") as f:
    state_of_the_union = f.read()
```

要使用 @[CharacterTextSplitter] 分割文本并使用 `tiktoken` 合并块，请使用其 `.from_tiktoken_encoder()` 方法。注意，此方法产生的分割可能大于 `tiktoken` 分词器计算的块大小。

`.from_tiktoken_encoder()` 方法接受 `encoding_name`（如 `cl100k_base`）或 `model_name`（如 `gpt-4`）作为参数。所有额外参数如 `chunk_size`、`chunk_overlap` 和 `separators` 用于实例化 `CharacterTextSplitter`：


```python
text_splitter = CharacterTextSplitter.from_tiktoken_encoder(
    encoding_name="cl100k_base", chunk_size=100, chunk_overlap=0
)
texts = text_splitter.split_text(state_of_the_union)
```


```python
print(texts[0])
```
```text
Madam Speaker, Madam Vice President, our First Lady and Second Gentleman. Members of Congress and the Cabinet. Justices of the Supreme Court. My fellow Americans.

Last year COVID-19 kept us apart. This year we are finally together again.

Tonight, we meet as Democrats Republicans and Independents. But most importantly as Americans.

With a duty to one another to the American people to the Constitution.
```
To implement a hard constraint on the chunk size, we can use `RecursiveCharacterTextSplitter.from_tiktoken_encoder`, where each split will be recursively split if it has a larger size:

要对块大小实施硬性约束，可以使用 `RecursiveCharacterTextSplitter.from_tiktoken_encoder`，如果某个分割超出大小限制，它将被递归地再次分割：


```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

text_splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
    model_name="gpt-4",
    chunk_size=100,
    chunk_overlap=0,
)
```

我们还可以加载 `TokenTextSplitter` 分割器，它直接与 `tiktoken` 配合工作，确保每个分割小于块大小。


```python
from langchain_text_splitters import TokenTextSplitter

text_splitter = TokenTextSplitter(chunk_size=10, chunk_overlap=0)

texts = text_splitter.split_text(state_of_the_union)
print(texts[0])
```
```text
Madam Speaker, Madam Vice President, our
```
某些书写语言（如中文和日文）的字符会被编码为两个或更多个 Token。直接使用 `TokenTextSplitter` 可能会将一个字符的 Token 分割到两个块中，导致 Unicode 字符格式错误。请使用 `RecursiveCharacterTextSplitter.from_tiktoken_encoder` 或 `CharacterTextSplitter.from_tiktoken_encoder` 来确保块包含有效的 Unicode 字符串。

```ts
import { TokenTextSplitter } from "@langchain/textsplitters";

const stateOfTheUnion = readFileSync("state_of_the_union.txt", "utf8");
```

要使用 @[TokenTextSplitter] 分割文本并使用 `tiktoken` 合并块，请在初始化 @[TokenTextSplitter] 时传入 `encodingName`（如 cl100k_base）。注意，此方法产生的分割可能大于 `tiktoken` 分词器计算的块大小。

```ts

// 示例：使用 cl100k_base 编码
const splitter = new TokenTextSplitter({ encodingName: "cl100k_base", chunkSize: 10, chunkOverlap: 0 });

const texts = splitter.splitText(stateOfTheUnion);
console.log(texts[0]);
```

## SentenceTransformers

[SentenceTransformersTokenTextSplitter](https://python.langchain.com/api_reference/text_splitters/sentence_transformers/langchain_text_splitters.sentence_transformers.SentenceTransformersTokenTextSplitter.html) 是一个专用于 sentence-transformer 模型的文本分割器。其默认行为是将文本分割为符合你所使用的 sentence-transformer 模型 Token 窗口大小的块。

要根据 sentence-transformers 分词器分割文本并约束 Token 数量，请实例化 `SentenceTransformersTokenTextSplitter`。你可以选择性地指定：

- `chunk_overlap`：Token 重叠数量；
- `model_name`：sentence-transformer 模型名称，默认为 `"sentence-transformers/all-mpnet-base-v2"`；
- `tokens_per_chunk`：每个块所需的 Token 数。


```python
from langchain_text_splitters import SentenceTransformersTokenTextSplitter

splitter = SentenceTransformersTokenTextSplitter(chunk_overlap=0)
text = "Lorem "

count_start_and_stop_tokens = 2
text_token_count = splitter.count_tokens(text=text) - count_start_and_stop_tokens
print(text_token_count)
```
```text
2
```

```python
token_multiplier = splitter.maximum_tokens_per_chunk // text_token_count + 1

# `text_to_split` 无法放入单个块中
text_to_split = text * token_multiplier

print(f"tokens in text to split: {splitter.count_tokens(text=text_to_split)}")
```
```text
tokens in text to split: 514
```

```python
text_chunks = splitter.split_text(text=text_to_split)

print(text_chunks[1])
```
```text
lorem
```

## KoNLPY

<Note>
[KoNLPy: 韩语自然语言处理 Python 库](https://konlpy.org/en/latest/) 是一个用于韩语自然语言处理（NLP）的 Python 包。
</Note>

Token 分割涉及将文本分割为更小、更易管理的单元（称为 Token）。这些 Token 通常是单词、短语、符号或其他有意义的元素，对后续的处理和分析至关重要。在英语等语言中，Token 分割通常通过空格和标点符号分隔单词。Token 分割的效果很大程度上取决于分词器对语言结构的理解，确保生成有意义的 Token。由于专为英语设计的分词器无法理解其他语言（如韩语）的独特语义结构，因此无法有效地用于韩语处理。

### 使用 KoNLPy 的 kkma 分析器进行韩语 Token 分割
对于韩语文本，KoNLPY 包含一个名为 `Kkma`（Korean Knowledge Morpheme Analyzer，韩语知识词素分析器）的形态学分析器。`Kkma` 提供韩语文本的详细形态学分析，将句子分解为单词，并将单词分解为各自的词素，识别每个 Token 的词性。它可以将一段文本分割为单独的句子，这对于处理长文本特别有用。

### 使用注意事项
虽然 `Kkma` 以其详细的分析而闻名，但需要注意的是，这种精确度可能会影响处理速度。因此，`Kkma` 最适合用于优先考虑分析深度而非快速文本处理的应用。


```python
# pip install konlpy
```


```python
# 这是一个我们想要分割的韩语长文档。
with open("./your_korean_doc.txt") as f:
    korean_document = f.read()
```


```python
from langchain_text_splitters import KonlpyTextSplitter

text_splitter = KonlpyTextSplitter()
```


```python
texts = text_splitter.split_text(korean_document)
# 句子通过 "\n\n" 字符进行分割。
print(texts[0])
```

## Hugging Face 分词器

[Hugging Face](https://huggingface.co/docs/tokenizers/index) 拥有许多分词器。

我们使用 Hugging Face 分词器 [GPT2TokenizerFast](https://huggingface.co/Ransaka/gpt2-tokenizer-fast) 来按 Token 数计算文本长度。

1. 文本如何分割：通过传入的字符。
2. 块大小如何计算：通过 `Hugging Face` 分词器计算的 Token 数。


```python
from transformers import GPT2TokenizerFast

tokenizer = GPT2TokenizerFast.from_pretrained("gpt2")
```


```python
# 这是一个可以被分割的长文档。
with open("state_of_the_union.txt") as f:
    state_of_the_union = f.read()
from langchain_text_splitters import CharacterTextSplitter
```


```python
text_splitter = CharacterTextSplitter.from_huggingface_tokenizer(
    tokenizer, chunk_size=100, chunk_overlap=0
)
texts = text_splitter.split_text(state_of_the_union)
```


```python
print(texts[0])
```
